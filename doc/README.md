# 使用说明

## 目录用途

本目录用于两类场景：

- 作为学员生成 Demo 项目的文档脚手架
- 作为 AI 初始化和实现项目时的上下文入口

## 推荐阅读顺序

1. `01-AI入口.md`
2. `02-需求和项目介绍/README.md`
3. `03-技术架构设计/README.md`
4. `04-功能详细设计/README.md`
5. `06-开发参考/README.md`
6. `07-开发规范/README.md`
7. `05-测试用例设计/README.md`
8. `08-项目进度/README.md`

## 使用建议

- 学员应先完善需求、架构、功能设计，再开始编码
- AI 应先读取入口文档和相关设计文档，再生成代码或测试
- 如项目发生变化，应优先更新设计文档，再同步更新进度和测试用例

## 维护原则

- 需求类文档描述做什么
- 设计类文档描述怎么做
- 参考类文档提供外部依赖和已有能力说明
- 规范类文档定义必须遵守的约束
