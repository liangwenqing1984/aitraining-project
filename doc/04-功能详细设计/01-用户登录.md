# 用户登录

## 功能目标

实现基于 OAuth2 授权码模式的用户认证，集成第三方 Auth 服务（leaf-auth-server），支持登录、Token 管理、Token 刷新、登出全流程。

## 页面或接口说明

- **登录页**：`/login` (`code/frontend/src/views/Login.vue`)
  - 页面加载时自动调用 `/api/auth/authorize-url` 获取授权 URL
  - 用户点击"登录"按钮 → 跳转到 Auth 服务登录页
  - 无用户名密码输入框（由 Auth 服务提供）
- **回调处理**：`/api/auth/callback?code=xxx&state=xxx`
  - 后端用 code 交换 Access Token + Refresh Token
  - 获取用户信息并存入 Cookie（`oauth_user_info`）
  - 重定向到系统主页
- **路由守卫**：`code/frontend/src/router/index.ts`
  - `beforeEach` 检查 Cookie 中是否有 `oauth_access_token`
  - 无 Token → 重定向到 `/login`
  - 有 Token → 放行

## 输入输出

- 输入：用户在 Auth 服务页面输入账号密码（不在本项目范围内）
- 输出：
  - Cookie: `oauth_access_token`、`oauth_refresh_token`（httpOnly）、`oauth_token_type`、`oauth_expires_in`、`oauth_user_info`
  - 重定向到系统主页，显示用户姓名

## 业务规则

- Auth 服务配置：
  - clientId: `aitraining`
  - clientSecret: `29b9635df0164eb890d99a58ffa7f8f2`
  - 授权范围: `openid all`
  - Token 交换使用 `multipart/form-data` 格式提交
- Token 生命周期：
  - Access Token 过期时间由 Auth 服务决定（通常 1800 秒）
  - Refresh Token 用于续期，通过 `/api/auth/refresh-token` 接口
  - 前端 Cookie 的 maxAge 与 Token expires_in 一致
- 登出流程：
  - 调用 `/api/auth/logout` → 后端调用 Auth 服务获取 logoutTicket → 清除所有 Cookie → 返回 logoutTicket 给前端
  - 前端重定向到 Auth 服务登出页（带 logoutTicket 参数）

## 异常处理

- Token 交换失败（code 过期/无效）：返回错误信息，前端显示"登录失败"提示，用户需重新发起登录
- Token 刷新失败：清除 Cookie，跳转登录页
- Auth 服务不可达：前端显示"认证服务暂不可用"，提示用户检查网络
- 用户信息获取失败：Token 仍有效但无法获取用户详情时，标记为降级登录状态

## 验收标准

- [ ] 未登录用户访问任意页面 → 自动跳转到登录页
- [ ] 点击登录 → 跳转到 Auth 服务授权页面
- [ ] 授权成功后 → 自动回调 → 显示系统主页 + 侧边栏用户信息
- [ ] Token 过期后 → 自动刷新（或跳转登录页）
- [ ] 点击登出 → 所有 Cookie 清除 → 跳转到登录页
- [ ] 关闭浏览器后重新打开 → Cookie 有效期内可免登录
