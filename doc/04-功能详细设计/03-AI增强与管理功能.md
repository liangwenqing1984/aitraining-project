# AI 增强与管理功能

## 功能目标

实现四大 AI 能力：数据增强标准化、智能市场洞察、自然语言查询、AI 反爬对抗，以及统一的 LLM 配置管理。

## 各功能说明

### 1. 数据增强标准化 (`/api/llm/enrich/:taskId`)

**功能目标**：将爬虫原始数据中的非结构化字段（薪资文本、职位描述等）通过 LLM 转为结构化标准数据。

**处理流程**：
```
用户点击"AI 增强" → POST /api/llm/enrich/:taskId
  → 读取任务关联的 Excel 文件原始数据
  → 按 BATCH_SIZE=5 分批提交给 LLM
  → LLM 返回 JSON（薪资数字、技能列表、行业分类等）
  → 3 层 JSON 解析降级 + 最多 3 次重试
  → ON CONFLICT UPSERT 写入 job_enrichments 表
  → WebSocket 推送 enrichment:progress
  → 全部完成后推送 enrichment:completed
```

**增强维度**：薪资标准化（月薪上下限 + 年薪估算）、职位分类（L1/L2）、公司行业（14 类）、技能提取（必备/加分）、学历规范、经验年限、福利识别、工作模式

### 2. 智能市场洞察 (`/api/llm/insights/:fileId`)

**功能目标**：基于 AI 增强后的标准化数据，调用 LLM 生成深度分析报告。

**处理流程**：
```
POST /api/llm/insights/:fileId
  → 检查增强数据是否存在（job_enrichments 有数据才继续）
  → 聚合统计（薪资分布/职位分类/学历/经验/技能/行业/工作模式）
  → 将统计数据作为 context 注入 Prompt
  → 调用 LLM 生成报告（temperature=0.3, maxTokens=16384）
  → 解析 LLM 输出的 JSON（sections[] + chartsConfig[] + summary）
  → 存入 market_reports 表
  → WebSocket 推送 insights:completed
  → 前端 2s 轮询直到报告出现
```

**报告内容**：Markdown 正文（多 sections）、200 字摘要、ECharts 图表配置数组

### 3. 自然语言查询 (`/api/llm/query`)

**功能目标**：用户在对话界面输入中文问题，AI 自动转 SQL 查询并总结结果。

**处理流程**：
```
POST /api/llm/query { query: "薪资最高的10个岗位", taskId?: "xxx" }
  → 构建 Prompt（含表结构 + 用户问题）
  → LLM 生成 SQL
  → validateSQL() 安全校验（仅允许 SELECT）
  → limitSQL() 自动添加 LIMIT 500
  → 执行 SQL → 获取结果
  → LLM 总结结果（自然语言）
  → 保存到 saved_queries 表
  → 返回 { summary, data, sql, count }
```

**安全措施**：SQL 白名单（SELECT only）、危险关键字阻断（INSERT/UPDATE/DELETE/DROP/ALTER/CREATE/TRUNCATE/EXEC/CALL/COPY/GRANT）、结果限制 500 行、参数化查询

### 4. AI 反爬对抗 (`/api/llm/anti-crawl/*`)

**功能目标**：用 LLM 智能检测反爬措施并推荐应对策略。

**三个接口**：
- `classifyPage(html, url)` → 判断页面类型（正常/验证码/WAF/登录/错误/空白）
- `suggestSelectors(html, target)` → 当选择器失效时，分析 HTML 推荐新选择器
- `recommendAction(classification)` → 根据页面类型推荐应对操作（继续/等待/重试/切换IP/验证码处理/中止）

**集成方式**：在爬虫页面加载后调用 classifyPage（5 秒冷却防频繁），在 DOM 解析异常时调用 suggestSelectors

### 5. LLM 配置管理 (`/api/llm/config`)

**功能目标**：用户在前端配置多个 LLM 模型，并为不同任务类型分配模型。

**配置项**：提供商（openai/deepseek/zhipu/ollama/anthropic）、模型名称、API Key（AES-256-GCM 加密）、Base URL、任务路由（JSONB 数组，如 `["enrichment", "insights"]`）

**内置 Provider**：
- CloudProvider：OpenAI 兼容 API（统一处理 DeepSeek/OpenAI/Zhipu/Anthropic）
- LocalProvider：Ollama 本地 API（`http://localhost:11434/api/generate`）

## 业务规则

- LLM 增强批处理：BATCH_SIZE=1 串行处理，间隔 500ms（避免 API 限流）
- 增强重试：单条记录最多 3 次尝试（间隔 1s），失败则跳过并计数
- JSON 解析降级：标准解析 → 正则清理（去注释、去控制字符）→ 子串提取 → 单引号转双引号
- Task Router：根据任务类型自动选择已配置的模型（优先匹配 `task_routing` 配置）
- 反爬冷却：同一页面 5 秒内不重复调用 classifyPage
