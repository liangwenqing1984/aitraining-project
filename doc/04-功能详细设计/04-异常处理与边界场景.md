# 异常处理与边界场景

## 通用异常

### 参数缺失
- 创建任务时缺少关键词或城市 → 后端返回 400 `{ success: false, error: "关键词和城市不能为空" }`
- AI 增强时 taskId 不存在 → 返回 404 `{ error: "任务不存在" }`
- 生成洞察时 fileId 不存在 → 返回 404 `{ error: "文件不存在" }`

### 鉴权失败
- 未登录（无 Cookie/Token 失效）→ 返回 401 → 前端路由守卫跳转 `/login`
- Token 过期 → 自动调用 `/api/auth/refresh-token` 续期 → 续期失败则清除 Cookie 跳转登录
- API Key 解密失败 → 记录错误日志 → 返回 `{ error: "LLM 配置异常，请检查 API Key" }`

### 外部服务异常
- 数据库不可用 → 连接池报错 → 返回 500 → 任务自动暂停
- Auth 服务不可达 → 登录/刷新 Token 失败 → 返回 `{ error: "认证服务暂不可用" }` → 前端显示提示
- LLM API 超时（默认 120s）→ 自动重试（最多 3 次）→ 仍失败则返回 `{ error: "LLM 调用超时" }`
- Puppeteer 浏览器崩溃 → 捕获 `browser disconnected` 异常 → 任务标记 `failed` → 写入 error_message

## 边界场景

### 空数据
- 搜索关键词无结果（0 条职位）→ 任务仍标记 `completed`，record_count=0，生成空 Excel（仅表头）
- AI 增强时 job_enrichments 无数据 → 洞察/查询接口提示"请先进行数据增强"
- 自然语言查询结果为空 → 返回 `{ summary: "查询无匹配结果", data: [], count: 0 }`

### 重复提交
- 重复启动同一任务 → 后端检查任务状态，若已 `running` 则返回 `{ error: "任务已在运行中" }`
- 重复 AI 增强 → 支持幂等操作：ON CONFLICT UPSERT 覆盖更新（不产生重复记录）
- 重复生成洞察 → 每次生成新报告（不覆盖历史），旧报告可通过 history 接口查看

### 并发冲突
- 多个任务同时操作同一数据库连接 → pg Pool (max 50) 自动排队，超时 5s 后报错
- 同时停止和启动任务 → stop 优先执行，完成后再处理 start
- WebSocket 多客户端订阅同一任务 → Socket.IO room 模式，所有订阅者同时收到推送

### 大文件处理
- Excel 行数超限 → 单个文件默认无行数限制（ExceJS 流式写入）
- NL 查询结果超 500 行 → `limitSQL()` 自动截断，前端提示"仅显示前 10 条，共 500 条结果"
- LLM 返回内容超 maxTokens → Token 被截断，可能导致 JSON 不完整 → 触发 JSON 解析降级 → 若无法修复则跳过

## 处理原则

- **快速失败**：参数校验在 Controller 层完成，不合法请求不进 Service 层
- **优雅降级**：LLM JSON 解析致命错误时才跳过单条记录，不中止整批
- **可恢复**：任务异常中止后，用户可从断点恢复（通过 Excel 中的已采集 jobId 集合）
- **可观测**：所有异常通过 WebSocket `task:error` 推送 + `tasks.error_message` 持久化
- **用户友好**：前端错误提示使用中文描述，不暴露技术细节（如 SQL 错误原文）
