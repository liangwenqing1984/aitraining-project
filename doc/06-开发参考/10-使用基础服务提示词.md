# 使用基础服务提示词

本文档整理一些可直接发给 AI 的“用户提示词”，用于让 AI 帮助生成接入基础服务的页面、接口封装、后端逻辑与联调代码。

使用这些提示词前，建议先让 AI 参考以下文档：

- [07-OAUTH2授权码模式对接说明.md](./07-OAUTH2授权码模式对接说明.md)
- [06-东方金信基础服务调用指南.md](./06-东方金信基础服务调用指南.md)
- [03-Base服务接口说明.md](./03-Base服务接口说明.md)

提示词中的 `{appCode}`、`{moduleCode}`、`{tenantId}`、`{baseHost}`、`{authHost}`、`{clientId}`、`{clientSecret}`、`{redirectUri}` 等占位符，请替换成项目实际值。

---

## 1. 对接基础服务 OAuth2，使用授权码模式

请帮我在当前项目中接入基础服务 OAuth2 登录，使用授权码模式。请严格参考 `doc/06-开发参考/07-OAUTH2授权码模式对接说明.md`，实现：

- 登录按钮跳转认证中心授权地址
- 回调页接收 `code` 和 `state`
- 服务端调用 `/auth2/oauth2/token` 用 `authorization_code` 换取 token
- 服务端保存 token，前端不暴露 `client_secret`
- 后续请求自动携带登录态
- 提供获取当前登录用户信息能力
- 处理 `state` 校验、授权失败、token 失效等异常

请直接输出完整代码，不要只给思路。

## 2. 对接基础服务 Base，使用文件上传下载能力

请帮我在当前项目中接入 BASE 文件能力，参考 `doc/06-开发参考/03-Base服务接口说明.md`，实现：

- `POST /file/upload` 单文件上传
- `POST /file/batch/upload` 多文件上传
- `GET /file/download` 文件下载
- `GET /file/info` 文件详情
- `GET /file/delete` 文件删除

要求生成一个文件管理页面，展示文件列表并支持上传、下载、删除；上传使用 `multipart/form-data`，带上 `appCode`，请求时保留登录态，接口失败时要有明确提示。请直接输出完整代码。

请帮我在当前项目中接入 BASE 工作流审批能力，参考 `doc/06-开发参考/03-Base服务接口说明.md` 和 `doc/06-开发参考/06-东方金信基础服务调用指南.md`，实现：

- `POST /workflow/node/complete` 审批通过

请生成审批列表和详情页面，处理分页、空状态、错误提示和通过/驳回后的刷新逻辑，直接输出完整代码。

## 4. 对接基础服务调度能力

请帮我在当前项目中接入 BASE 调度能力，参考 `doc/06-开发参考/03-Base服务接口说明.md` 和 `doc/06-开发参考/06-东方金信基础服务调用指南.md`，实现：

- `GET /job/group/pageJobGroup` 分页列表
- `POST /job/group/addSimpleJobGroup` 新建简单任务组
- `POST /job/group/start` 启动
- `POST /job/group/stop` 停止
- `POST /job/group/startSchJobGroup` 测试执行
- `POST /job/group/delete` 删除

请生成一个调度任务管理页面，支持分页查询、创建、启动、停止、测试执行和删除；注意 `grpId` 与 `jobGroupId` 参数区别，操作成功后刷新列表，并对失败情况给出明确提示。请直接输出完整代码。
