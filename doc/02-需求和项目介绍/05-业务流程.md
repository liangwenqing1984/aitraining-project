# 业务流程

## 主流程

1. **用户登录** → 浏览器访问系统 → 重定向到 OAuth2 授权页面 → 输入账号密码 → 授权码回调 → 换取 Access Token → 存入 Cookie → 返回系统主页
2. **创建采集任务** → 选择平台（智联/前程无忧/全部）→ 输入关键词并添加到列表 → 选择目标城市（多选）→ 设置最大页数 → 创建任务 → 任务状态 `pending`
3. **执行采集** → 点击"启动" → 状态变为 `running` → Puppeteer 启动浏览器 → 自动搜索关键词 → 逐页抓取职位列表 → 逐个打开详情页抓取 → WebSocket 实时推送进度和日志 → 遇到反爬自动降级串行 → 完成后生成 Excel 文件 → 状态 `completed`
4. **AI 数据增强** → 点击任务的"AI 增强"按钮 → 系统按批（每批 5 条）将原始职位数据提交给 LLM → LLM 返回结构化 JSON → 解析后写入 `job_enrichments` 表 → WebSocket 推送增强进度 → 全部完成后通知
5. **AI 深度分析** → 进入智能分析页 → 选择文件 → 系统检查增强数据是否就绪 → 聚合薪资/城市/学历/技能/行业等多维度统计 → 调用 LLM 生成报告 → 轮询等待报告完成 → 展示 Markdown 内容 + ECharts 图表
6. **自然语言查询** → 进入智能查询页 → 输入中文问题 → LLM 生成参数化 SQL → 安全白名单校验 → 执行查询 → LLM 总结结果 → 展示表格 + 自然语言回答 + SQL

## 关键分支流程

- 分支1（断点续传）：任务启动时 → 检查是否存在 Excel 文件 → 读取已采集的 jobId 集合 → 搜索页滚动到上次中断位置 → 跳过已采集的职位 → 增量追加新数据
- 分支2（WAF 反爬降级）：页面加载后 → AI 分类页面类型 → 检测到 WAF 拦截 → 自动降低并发数 → 增加随机延迟（2-5 秒）→ 随机化浏览器指纹 → 重试 → 若持续失败则标记任务 `failed`
- 分支3（LLM 增强重试）：LLM 调用失败 → 记录警告 → 等待 1 秒 → 重试（最多 3 次）→ 若 JSON 解析失败 → 使用 3 层降级解析（正则清理→子串提取→单引号转双引号）→ 仍失败则跳过该条记录并计数

## 异常流程

- 异常场景1：浏览器崩溃（内存不足 / Chrome 进程被杀）→ Puppeteer 抛出 `browser disconnected` 错误 → 任务状态标记 `failed` → 用户可点击"恢复"从断点继续
- 异常场景2：LLM API 返回空内容 → 系统抛出"LLM 返回空内容"错误 → 增强/报告任务标记当前批次失败 → 用户可重新触发
- 异常场景3：数据库连接池耗尽 → 等待连接超时 → 自动释放空闲连接 → 若仍失败则任务暂停 → 需人工检查数据库负载
- 处理方式：所有异常通过 WebSocket 推送 `task:error` 事件 → 前端在监控页实时展示红色错误日志行 → 严重错误同时写 `tasks.error_message` 字段供后续排查
