# 用户与场景

## 目标用户

- 用户类型：
  - **HR 分析师**：需要市场薪资数据支撑招聘策略和薪酬体系设计
  - **技术管理者**：需要追踪技术岗位的技能需求变化趋势
  - **招聘顾问**：需要快速获取多城市、多岗位的市场数据对比
  - **AI 全栈开发学员**：学习企业级全栈项目开发，了解 AI 集成实践
- 用户特征：
  - 熟悉业务需求但不一定有技术背景（需要自然语言查询界面）
  - 可能需要批量处理大量数据（需要批量任务创建和管理）
  - 对 AI 模型有一定了解（需要灵活的模型配置能力）

## 核心使用场景

- 场景1（数据采集）：用户选择关键词"Java开发"、城市"北京/上海/深圳"，创建批量采集任务 → 系统自动爬取职位列表和详情 → 实时查看进度和分级日志 → 数据导出为 Excel 文件
- 场景2（AI 数据增强）：任务采集完成 → 用户点击"AI 增强"按钮 → LLM 自动将原始薪资文本转为标准化数字、提取技能列表、分类职位和行业 → 增强进度实时推送 → 增强完成后在分析页面查看标准化数据
- 场景3（智能分析与查询）：用户在智能分析页点击"AI 深度分析" → 系统聚合统计数据并调用 LLM 生成 Markdown 报告 + ECharts 图表 → 或用户在自然语言查询页输入"薪资最高的10个岗位" → AI 转 SQL 查询并总结结果

## 用户关键诉求

- 数据采集要稳定可靠，反爬能力强，支持断点续传
- 数据标准化要准确，避免 AI 猜测导致错误分类
- 查询结果要快速（Text-to-SQL 毫秒级执行）且安全（仅允许 SELECT）
- 报告要有可读性，图表能直观展示趋势
- 模型配置要灵活，支持按任务类型分配不同模型
