# 异常场景测试

> 目的：验证系统在异常条件下的容错和恢复能力。

| 编号 | 场景 | 前置条件 | 步骤 | 预期结果 |
| --- | --- | --- | --- | --- |
| EX-01 | 搜索无结果 | 任务 running | 使用不存在的关键词（如"xzypq123"）启动采集 | 任务完成，record_count=0，Excel 仅含表头 |
| EX-02 | 浏览器崩溃恢复 | 任务 running | 模拟关闭 Chrome 进程 → 观察任务状态 | 任务状态变为 failed，error_message 记录"browser disconnected" |
| EX-03 | 浏览器崩溃后重试 | 任务 failed（EX-02 后） | 点击"恢复" | 从断点继续采集，日志显示已跳过数量 |
| EX-04 | LLM 增强-API Key 错误 | 已配置 LLM，API Key 不正确 | 启动数据增强 | 增强失败，状态返回 error，提示检查 API Key |
| EX-05 | LLM 增强-模型超时 | 已配置 LLM，设置极短超时 | 启动数据增强 | 自动重试 3 次 → 仍失败则跳过该条记录 → 继续处理下一条 |
| EX-06 | LLM 增强-JSON 格式异常 | LLM 返回非标准 JSON | 观察后端日志 | 触发 3 层降级解析（正则清理→子串提取→引号转换）→ 成功或跳过 |
| EX-07 | LLM 返回空内容 | 生成洞察时 LLM 返回空字符串 | 调用生成洞察 | 返回 error"LLM 返回空内容，报告生成失败" |
| EX-08 | NL 查询-危险 SQL | 已登录 | 尝试通过各种方式让 LLM 生成 UPDATE/DELETE 语句 | validateSQL() 拒绝执行，返回"不允许的查询" |
| EX-09 | NL 查询-无关问题 | 已登录 | 输入"今天天气怎么样" | LLM 生成的 SQL 查询无结果或返回空 → 友好提示 |
| EX-10 | 数据库连接中断 | 任务 running | 模拟数据库断连（停止 PostgreSQL） | 任务暂停或失败，前端显示"数据库连接异常" |
| EX-11 | 数据库恢复后重连 | EX-10 后 | 恢复数据库连接 → 重启任务 | 连接池自动恢复，任务可从断点继续 |
| EX-12 | 并发请求压测 | 多个用户同时操作 | 快速连续创建 10 个任务 + 同时启动 | 连接池排队处理，无死锁，无数据丢失 |
| EX-13 | 大结果集查询 | 增强数据 1000+ 条 | NL 查询"列出所有岗位" | SQL 自动添加 LIMIT 500，前端提示"仅显示前 10 条，共 500 条" |
| EX-14 | 文件上传边界 | 已登录 | 尝试上传非 Excel 文件（如果存在上传功能） | 拒绝处理，提示"仅支持 .xlsx 格式" |
| EX-15 | XSS 注入尝试 | 已登录 | 在任务名称中输入 `<script>alert(1)</script>` | Vue 模板绑定自动转义，不执行脚本 |
| EX-16 | SQL 注入尝试 | NL 查询页 | 输入"'; DROP TABLE tasks; --" | LLM 不生成恶意 SQL；即使生成，validateSQL() 拒绝执行 |
| EX-17 | 反爬-WAF 拦截 | 爬虫任务 running | 智联返回 WAF 页面（模拟或真实） | AI 分类检测到 WAF → 自动降级串行 + 增加延迟 |
| EX-18 | 反爬-连续拦截 | EX-17 持续 | 串行模式仍被拦截 | 任务标记高风险，日志提示"建议更换 IP 或稍后重试" |
