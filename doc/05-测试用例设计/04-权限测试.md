# 权限测试

> 目的：验证认证与授权边界的正确性。

| 编号 | 场景 | 前置条件 | 步骤 | 预期结果 |
| --- | --- | --- | --- | --- |
| AUTH-01 | 未登录访问首页 | Cookie 中无 oauth_access_token | 直接访问 `http://localhost:3000/` | 自动跳转到 `/login` |
| AUTH-02 | 未登录访问 API | 无 Token | 直接请求 `GET /api/tasks` | 返回 401 Unauthorized |
| AUTH-03 | 已登录访问页面 | Cookie 中有有效 Token | 访问 `/crawler` | 正常渲染，不跳转登录页 |
| AUTH-04 | 已登录调用 API | Cookie 中有有效 Token | 请求 `GET /api/tasks` | 返回 200 + 数据 |
| AUTH-05 | Token 过期后访问 | Cookie 中有过期 Token（可模拟设置短过期时间） | 访问 `/crawler` | 前端自动尝试刷新 Token → 成功则继续，失败则跳转登录 |
| AUTH-06 | Token 过期后 API | 过期 Token | 请求 `GET /api/tasks` | 返回 401 → 前端触发刷新流程 |
| AUTH-07 | Refresh Token 也过期 | 两个 Token 均过期 | 访问任意页面 | 跳转登录页，所有 Cookie 被清除 |
| AUTH-08 | 登出后访问 | 已点击登出 | 访问 `/crawler` | 跳转到登录页，Cookie 中无 Token |
| AUTH-09 | 登出后直接访问 API | 已登出，手动构造请求 | 用浏览器 DevTools 发送带旧 Token 的请求 | 返回 401（旧 Token 已失效） |
| AUTH-10 | 伪造 Token 访问 API | 手动设置 Cookie | 设置 `oauth_access_token=invalid_token_string` | 返回 401，Token 验证失败 |
| AUTH-11 | 删除 Cookie 后的行为 | Cookie 被手动清除 | 清除浏览器 Cookie → 刷新页面 | 跳转到登录页 |
