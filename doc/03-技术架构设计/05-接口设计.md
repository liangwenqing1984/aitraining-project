## 请求与响应约定

- **请求头约定**：
  - `Content-Type: application/json`（普通 JSON 请求）
  - `Authorization: Bearer <access_token>`（需要认证的接口）
  - `Cookie: oauth_access_token=<token>`（前端自动携带）
- **统一响应结构**：
  ```json
  {
    "success": true,
    "data": { /* 业务数据 */ },
    "error": "错误描述（仅失败时）"
  }
  ```
- **错误处理约定**：
  - 400：参数校验失败
  - 401：未认证 / Token 过期
  - 404：资源不存在
  - 500：服务端异常（错误详情通过 `error` 字段返回）

## 外部服务依赖

- **Auth 服务**：`leaf-auth-server.dev.jinxin.cloud`
  - 授权地址：`/auth2/oauth2/authorize`（授权码模式）
  - Token 交换：`/auth2/oauth2/token`
  - 用户信息：`/auth2/api/v2/user/getLoginUser`
  - 登出：`/auth2/api/v2/login/logoutTicket`
  - 详细接口说明见 `../06-开发参考/02-Auth服务接口说明.md`
- **Base 服务**：未直接调用，基础服务能力见 `../06-开发参考/03-Base服务接口说明.md`
- **LLM API**：根据配置动态选择提供商
  - DeepSeek: `https://api.deepseek.com/v1`
  - OpenAI: `https://api.openai.com/v1`
  - Anthropic: 通过 OpenAI 兼容代理
  - 智谱 GLM: `https://open.bigmodel.cn/api/paas/v4`
  - Ollama: `http://localhost:11434/api`
