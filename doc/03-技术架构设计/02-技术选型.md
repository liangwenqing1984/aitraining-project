# 技术选型

## 技术栈

| 层级 | 技术 | 版本要求 |
|------|------|----------|
| 前端框架 | Vue 3 (Composition API) | 3.5 |
| UI 组件库 | Element Plus | 最新 |
| 状态管理 | Pinia | 最新 |
| 路由 | Vue Router 4 | 4.x |
| 图表 | ECharts | 6 |
| 构建工具 | Vite | 5.x |
| HTTP 客户端 | Axios | 最新 |
| WebSocket | Socket.IO Client | 最新 |
| Markdown 渲染 | marked.js | 内嵌 |
| 后端框架 | Express | 4.x |
| 语言 | TypeScript | 5.3 |
| 运行时 | Node.js | >= 18.0 |
| 数据库驱动 | pg (node-postgres) | 最新 |
| 浏览器自动化 | Puppeteer | 最新 |
| Excel 读写 | ExcelJS | 最新 |
| 加密 | Node.js crypto (AES-256-GCM) | 内置 |
| 数据库 | PostgreSQL (SeaboxSQL) | >= 14 |
| 认证 | OAuth2 授权码模式 | - |
| 本地 LLM | Ollama | 最新 |
| 云端 LLM | DeepSeek / OpenAI / Anthropic / 智谱 | - |
| 包管理器 | npm | >= 9 |

## 选型理由

- **Vue 3 + Element Plus**：组合式 API 提供更好的 TypeScript 支持和逻辑复用，Element Plus 提供成熟的企业级后台 UI 组件
- **Express + TypeScript**：轻量灵活，与 Node.js 生态无缝集成，TypeScript 提供类型安全
- **PostgreSQL (SeaboxSQL)**：项目指定数据库，支持 JSONB 字段（适合 AI 增强的结构化数据），连接池最大 50
- **Puppeteer**：完整 Chrome 浏览器自动化，支持反爬场景（比 Playwright 更成熟），自动管理 Chromium 下载
- **Socket.IO**：成熟的 WebSocket 封装，支持房间（room）模式按 taskId 推送，自动降级长轮询
- **Ollama**：本地 LLM 部署方案，数据不出本地，适合批量数据增强（成本低），支持 Qwen/Llama 等开源模型
- **AES-256-GCM**：Node.js 内置 crypto 模块，认证加密模式，适合 API Key 的安全存储

## 关键技术决策

| 决策 | 选择 | 替代方案 | 理由 |
|------|------|----------|------|
| LLM 调用架构 | Provider 模式 + Task Router | 每个任务独立调用 | 统一接口，模型可互换，配置驱动 |
| 数据库 ORM | 原生 SQL (pg) | Sequelize/TypeORM | 灵活控制 SQL，JSONB 查询无学习成本 |
| Prompt 存储 | 代码常量 (prompts.ts) | 数据库 | 版本管理（Git），易于 review |
| 前端构建 | Vite | Webpack | 开发热更新快，Vue 3 官方推荐 |
| API Key 存储 | 数据库加密 (AES-256-GCM) | 环境变量 | 支持前端界面配置多个模型 |
| 反爬检测 | AI 分类 + 规则降级 | 纯规则 | AI 可识别未知的反爬页面类型 |
