# 系统模块设计

## 模块职责

### 数据采集模块 (`services/crawler/`)
- 智联招聘爬虫 `zhilian.ts`（~50KB）：关键词搜索 → 列表页解析 → 详情页抓取 → WAF 检测与降级 → 断点续传
- 前程无忧爬虫 `job51.ts`：同上流程
- 输出：AsyncGenerator<JobData>，通过 `yield` 逐条产出职位数据
- 公共接口：`async *crawl(config, signal): AsyncGenerator<JobData>`

### LLM 服务模块 (`services/llm/`)
- `index.ts`：LLMService 核心类 — 统一调用接口、Provider 注册、Task Router、重试/超时控制
- `prompts.ts`：所有 System/User Prompt 模板集中管理
- `enrichment.ts`：批量数据增强，每批 5-10 条，3 层 JSON 降级解析，3 次重试
- `insights.ts`：市场洞察 — 聚合统计 + LLM 报告生成 + WebSocket 进度推送
- `query.ts`：Text-to-SQL — NL 转 SQL + 安全白名单校验 + 结果总结
- `antiCrawl.ts`：页面分类 + 选择器推荐 + 应对策略推荐
- `providers/cloud.ts`：OpenAI 兼容 API 调用（支持所有云端提供商）
- `providers/local.ts`：Ollama 本地 API 调用

### 数据库模块 (`config/database.ts`)
- 连接池管理（max 50，超时 5s）
- 6 张表自动迁移（CREATE TABLE IF NOT EXISTS）
- Snake_case → camelCase 自动转换
- SQLite `?` 占位符 → PostgreSQL `$1` 占位符自动转换
- 封装 `db.prepare(sql).all()/.get()/.run()` 查询接口

## 模块交互

```
Frontend (Vue 3)
    │  HTTP REST (Axios) + WebSocket (Socket.IO Client)
    ▼
Express Routes (routes/)
    │  参数校验 + 响应封装
    ▼
Controllers (controllers/)
    │  业务编排
    ▼
Services (services/)
    │  ┌─────────┬──────────┬──────────┬──────────┐
    │  │TaskService│LLMService│AuthService│Crawler   │
    │  └─────────┴──────────┴──────────┴──────────┘
    │  SQL (db.prepare)
    ▼
Database (config/database.ts)
    │  pg Pool (10.1.1.113:7300)
    ▼
PostgreSQL (SeaboxSQL, schema: liangwenqing)
```

- 模块间调用关系：
  - `TaskService` → `Crawler` (启动/停止爬虫)
  - `TaskService` → `LLMService` (触发数据增强)
  - `Controllers` → `Services` (所有业务操作)
  - `Socket` → `io.emit()` 推送事件给前端
- 关键依赖：
  - LLM 模块依赖 `llm_config` 表获取模型配置和 API Key
  - 增强/洞察/查询模块依赖 `job_enrichments` 表作为数据源
  - 爬虫模块依赖 AI 反爬服务（可选，通过 LLM Service 调用）
