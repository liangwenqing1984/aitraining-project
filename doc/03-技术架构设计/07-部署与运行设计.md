# 部署与运行设计

## 本地运行方式

- **前端启动方式**：
  ```bash
  cd code/frontend
  npm install
  npm run dev
  # 默认端口 3000，端口被占用时自动 fallback 到 3002
  ```
- **后端启动方式**：
  ```bash
  cd code/backend
  npm install
  npm run dev
  # 使用 tsx watch 启动，端口 3004
  # 入口文件: src/index.ts
  ```
- **数据库准备方式**：
  - 无需手动建表，服务启动时自动执行 `initDatabase()` 迁移
  - 表结构通过 `CREATE TABLE IF NOT EXISTS` 自动创建
  - 字段变更通过 `information_schema.columns` 检测并 ALTER TABLE 添加
  - 索引自动创建（`CREATE INDEX IF NOT EXISTS`）
- **一键启动 (Windows)**：
  ```bash
  # 双击根目录 start-dev.bat
  # 自动清理 Node.js 缓存 → 启动后端 → 等待 3 秒 → 启动前端
  ```

## 环境说明

- **本地环境**：
  - OS：Windows 10/11 或 macOS/Linux
  - Node.js >= 18.0
  - Chrome/Chromium（Puppeteer 自动下载到 `node_modules/puppeteer/.local-chromium/`）
  - 网络要求：可访问 `10.1.1.113:7300`（数据库）和 `leaf-auth-server.dev.jinxin.cloud`（认证）
- **数据库环境**：
  - Host: `10.1.1.113`
  - Port: `7300`
  - Database: `training_exercises`
  - User/Password: `liangwenqing`
  - Schema: `liangwenqing`
  - 连接池: max 50, idle timeout 30s, connection timeout 5s
- **LLM 环境**（可选，按需配置）：
  - 云端 API：需要对应提供商的 API Key（在前端「AI 配置」页面填入）
  - 本地 Ollama：`http://localhost:11434`，需手动安装并拉取模型（如 `ollama pull qwen3:14b`）

## 配置项

- **必填环境变量**：
  - 无强制必填。数据库连接、OAuth2 凭证、加密密钥均有硬编码默认值（适用于本地开发和演示）。
- **可选环境变量**：
  - `AUTH_REDIRECT_URI`：OAuth2 回调地址（默认 `http://localhost:3000/api/auth/callback`）
  - `NODE_ENV`：`production` 时 Cookie secure=true，日志精简
  - `ENCRYPTION_KEY`：API Key 加密密钥（32 字节 hex），未设置时使用内置默认密钥
  - `PORT`：后端监听端口（默认 3004）

## 服务端口一览

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端 Vite Dev Server | 3000 (fallback 3002) | HMR 热更新 |
| 后端 Express API | 3004 | REST API + WebSocket |
| PostgreSQL (SeaboxSQL) | 7300 | 远程数据库 (10.1.1.113) |
| Ollama (本地) | 11434 | 本地 LLM 推理服务 |
| Auth 服务 | 80/443 | leaf-auth-server.dev.jinxin.cloud |
