# 数据库设计

## 文档定位

本文件描述当前项目的数据库设计结果（schema `liangwenqing`，共 6 张业务表 + 11 个索引）。

## 数据实体

- **Task**（爬虫任务）：采集任务的配置、状态、进度和产出
- **CsvFile**（导出文件）：任务完成后的 Excel 文件元数据
- **LLMConfig**（AI 模型配置）：LLM 提供商的连接信息和任务路由
- **JobEnrichment**（AI 增强结果）：AI 对原始职位数据的标准化结果
- **MarketReport**（市场洞察报告）：AI 生成的分析报告和图表配置
- **SavedQuery**（NL 查询历史）：用户的自然语言查询记录和结果

## 表设计概览

### 1. tasks — 爬虫任务

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(255) | PK | UUID |
| name | VARCHAR(500) | NOT NULL | 任务名称 |
| source | VARCHAR(50) | NOT NULL | zhilian / 51job / all |
| config | JSONB | NOT NULL | 关键词/城市/企业/页数配置 |
| status | VARCHAR(20) | DEFAULT 'pending' | pending→running→completed/failed |
| progress | INTEGER | DEFAULT 0 | 进度百分比 |
| total | INTEGER | DEFAULT 0 | 预计总数 |
| current | INTEGER | DEFAULT 0 | 当前完成数 |
| record_count | INTEGER | DEFAULT 0 | 采集记录数 |
| error_count | INTEGER | DEFAULT 0 | 错误计数 |
| error_message | TEXT | - | 严重错误详情 |
| csv_path | TEXT | - | 输出 Excel 文件路径 |
| start_time | TIMESTAMP | - | 任务开始时间 |
| end_time | TIMESTAMP | - | 任务结束时间 |
| created_at | TIMESTAMP | DEFAULT NOW | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT NOW | 更新时间 |

索引：`idx_tasks_status`、`idx_tasks_created_at`

## 关键设计说明

- **主键策略**：tasks/csv_files/job_enrichments/market_reports/saved_queries 使用 UUID (VARCHAR(255))，llm_config 使用 SERIAL 自增整数
- **租户策略**：当前单 schema (`liangwenqing`)，无多租户隔离
- **逻辑删除策略**：不保留已删除记录（物理删除：DELETE CASCADE）
- **审计字段策略**：所有表包含 `created_at`，tasks 和 llm_config 额外包含 `updated_at`
- **JSONB 列**：job_enrichments 使用 JSONB 存储技能和福利列表（支持索引查询），llm_config 使用 JSONB 存储任务路由配置
- **ON CONFLICT UPSERT**：job_enrichments 的 `UNIQUE(task_id, job_id)` 约束支持幂等重新增强（覆盖更新）

## 索引设计说明

- **高频查询场景**：`idx_tasks_status`（任务列表按状态筛选）、`idx_tasks_created_at`（任务列表按时间排序）
- **关联查询场景**：`idx_csv_files_task_id`（按任务查文件）、`idx_job_enrichments_task`（按任务查增强数据，增强/洞察/查询模块均依赖此索引）
- **报告查询场景**：`idx_market_reports_file`（按文件查报告历史）、`idx_saved_queries_task`（按任务查查询历史）
- **唯一约束场景**：`UNIQUE(task_id, job_id)` on job_enrichments
- **分页与排序场景**：`idx_tasks_created_at DESC`（任务列表按创建时间倒序）
