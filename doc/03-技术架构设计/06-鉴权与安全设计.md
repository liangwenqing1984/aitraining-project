# 鉴权与安全设计

## 鉴权方案

- **登录方式**：OAuth2 授权码模式（Authorization Code Grant）
  - 用户访问系统 → 后端生成 state 参数并返回授权 URL → 前端重定向到 `leaf-auth-server` 登录页
  - 用户输入账号密码 → 授权成功后回调 `/api/auth/callback?code=xxx&state=xxx`
  - 后端用 `code` + `client_id` + `client_secret` 交换 Access Token + Refresh Token
  - Token 写入 Cookie（`oauth_access_token`、`oauth_refresh_token`）
  - 前端路由守卫检查 Cookie 中是否存在 Token，不存在则跳转登录
- **Token 处理方式**：
  - Access Token 通过 `oauth_access_token` Cookie 传递给前端（httpOnly: false，便于显示用户信息）
  - Refresh Token 通过 `oauth_refresh_token` Cookie 存储（httpOnly: true，更安全）
  - Token 过期时，前端调用 `/api/auth/refresh-token` 自动续期
  - 登出时调用 `/api/auth/logout` 清除所有 Cookie + 获取 logoutTicket
- **会话或权限校验方式**：
  - 前端路由守卫 `router.beforeEach` 检查 Cookie 中是否有 access_token
  - 后端中间件验证 Token 有效性（调用 Auth 服务 `/api/v2/user/getLoginUser`）

## 安全要求

- **敏感信息处理**：
  - LLM API Key 使用 **AES-256-GCM** 加密存储于 `llm_config.api_key_encrypted` 字段
  - 加密密钥：`process.env.ENCRYPTION_KEY`（32 字节 hex）
  - 每次加密使用随机 16 字节 IV，输出格式：`iv:authTag:ciphertext`（hex 编码）
  - 前端显示 API Key 时自动脱敏（`sk-...xxxx`）
- **输入校验**：
  - SQL 注入防护：
    - 所有数据库操作使用 `$1, $2` 参数化查询（通过 `convertSqlPlaceholders` 自动转换）
    - NL 查询生成的 SQL 必须通过 `validateSQL()` 白名单校验：仅允许 SELECT 开头，阻断 INSERT/UPDATE/DELETE/DROP/ALTER/CREATE/TRUNCATE/EXEC/CALL/COPY/GRANT
    - 结果数量自动限制 500 行（`limitSQL()`）
  - XSS 防护：前端使用 Vue 模板绑定（自动转义 HTML），`v-html` 仅用于可信的 Markdown 渲染输出
- **日志脱敏**：
  - Token 交换日志仅输出前 10 字符（`code.substring(0, 10) + '...'`）
  - 错误日志不输出完整 API Key
  - 用户信息日志仅输出 userId、userName、cnName

## 认证流程示意

```
用户访问系统
  ↓
前端路由守卫检查 oauth_access_token Cookie
  ↓ 不存在
重定向到 /login → 跳转到 Auth 服务授权页
  ↓ 用户登录成功
回调 /api/auth/callback?code=xxx
  ↓
后端交换 Token → 写入 Cookie → 重定向到主页
  ↓
后续请求自动携带 Cookie → 后端验证 Token
  ↓ Token 过期
/api/auth/refresh-token → 使用 refresh_token 获取新 Token
```
