# 总体架构

## 架构概览

- 前端：Vue 3 (Composition API) + Element Plus + Pinia + ECharts 6 + Socket.IO Client
- 后端：Node.js + Express + TypeScript，端口 3004
- 数据库：PostgreSQL (SeaboxSQL)，10.1.1.113:7300，schema `liangwenqing`
- 外部依赖：OAuth2 认证服务 (leaf-auth-server)、Puppeteer (Headless Chrome)、LLM API (DeepSeek/OpenAI/Anthropic/智谱/Ollama)

## 关键设计原则

- **单一职责**：每个 Service 只负责一个领域（爬虫/LLM/认证），Controller 只做参数校验和响应封装
- **Provider 模式**：LLM 调用通过统一接口 `callLLM(system, user, options)` 抽象，底层 Provider（Cloud/Local）可互换
- **安全优先**：API Key AES-256-GCM 加密存储；SQL 仅允许 SELECT + 白名单校验；Token 通过 HttpOnly Cookie 传递
- **渐进式增强**：爬虫先尝试并发（默认并发数），遇到 WAF 自动降级串行；LLM JSON 解析 3 层降级
- **实时可观测**：所有长任务通过 WebSocket 推送进度（task:progress / enrichment:progress / insights:progress），日志按级别彩色输出
