## 💡 深层问题分析

### 为什么并发5必然失败？

#### A. **WAF的风控策略**

现代WAF（如Cloudflare、Akamai）通常采用多层风控：

1. **速率限制（Rate Limiting）**
   - 单IP每秒请求数阈值：通常3-5个
   - 超过阈值立即触发验证
   - 本次任务QPS=20（5个请求/250ms），远超阈值

2. **行为分析（Behavior Analysis）**
   - 检测请求的时间分布（是否均匀、是否有随机性）
   - 检测浏览器指纹（是否为真实浏览器）
   - 检测JavaScript执行环境（是否有异常）

3. **IP信誉度（IP Reputation）**
   - 历史行为记录（是否有过爬虫行为）
   - 数据中心IP vs 家庭宽带IP
   - 同一IP段的集体行为

#### B. **本次任务的致命缺陷**

1. **瞬时高峰**：5个请求在250ms内发出
   ```
   t=0ms:    请求1
   t=2ms:    请求2
   t=3ms:    请求3
   t=59ms:   请求4
   t=62ms:   请求5
   ```
   → WAF检测到**突发流量**，立即拦截

2. **缺乏随机性**：所有请求几乎同时发起
   - 人类浏览会有随机延迟（1-5秒）
   - 爬虫的请求间隔过于规律

3. **IP已被标记**：之前的测试让IP进入"观察名单"
   - 即使降低并发，仍处于高风险状态
   - 需要更长的"冷却时间"或更换IP
