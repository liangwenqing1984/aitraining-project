# 51job 输出字段优化与双源字段分离

## 涉及提交

| 提交 | 修复内容 |
|------|---------|
| `9210888` | 51job Excel输出适配目标20字段，智联保持原有字段和顺序 |

---

## 需求

前程无忧（51job）爬取结果的 Excel 字段和顺序需调整为目标格式：

```
公司名称 → 经营范围 → 公司规模 → 注册地址 → 工作地址 → 岗位名称 → 职能类别 → 职称分类 → 工作经验 → 学历 → 发布时间 → 薪资 → 工作类型 → 是否紧急招聘 → 职位描述 → 城市 → 企业类型 → 职位详情链接 → 公司详情链接 → 数据来源
```

同时智联招聘保持原有 19 个字段和顺序不变。两个数据源共用同一个 `JobData` 类型和 `TaskService` 写入逻辑，需在代码层面分离。

---

## 变更详情

### 1. JobData 类型新增 5 个字段

```typescript
// types/index.ts — 在原有19字段末尾追加
registeredAddress: string;  // 注册地址
titleCategory: string;      // 职称分类
isUrgent: string;           // 是否紧急招聘
jobDetailUrl: string;       // 职位详情链接
companyDetailUrl: string;   // 公司详情链接
```

向后兼容：原有字段顺序不变，5 个新字段追加在末尾。智联爬虫（`@ts-nocheck`）不设置这些字段时返回 `undefined`，写入 Excel 时转为空字符串。

### 2. 常量层分离两套字段集

```typescript
// constants.ts

// 智联（原有19字段）
CSV_FIELDS = ['企业名称', '职位ID', '职位名称', ...]
ZHILIAN_EXCEL_KEYS = ['companyName', 'jobId', 'jobName', ...]

// 前程无忧（新20字段 + 末尾jobId去重用）
CSV_FIELDS_51JOB = ['公司名称', '经营范围', '公司规模', '注册地址', ...]
JOB51_EXCEL_KEYS = ['companyName', 'businessScope', 'companyScale', 'registeredAddress', ...]
```

### 3. TaskService 按数据源自动切换

**创建 Excel** — 按首个站点选表头：
```typescript
private async createExcelFile(filepath: string, site?: string) {
  const is51job = site === '51job';
  const fields = is51job ? CSV_FIELDS_51JOB : CSV_FIELDS;
  // ...
}
```

**写入数据行** — 按 `job.dataSource` 动态选列序：
```typescript
const is51job = job.dataSource === '前程无忧';
const keys = is51job ? JOB51_EXCEL_KEYS : ZHILIAN_EXCEL_KEYS;
const rowValues = keys.map(key => (job as any)[key] || '');
```

**去重列检测** — 不再硬编码列号，动态查找"职位ID"列：
```typescript
headerRow.eachCell((cell, colNumber) => {
  if (cell.value?.toString()?.trim() === '职位ID') {
    jobIdColIndex = colNumber;
  }
});
```

### 4. 51job 数据提取三层填充

| 层级 | 提取方式 | 新字段来源 |
|------|---------|-----------|
| XHR拦截 | API JSON 响应 | `jobType`→titleCategory, `isUrgent`→isUrgent, `companyHref`→companyDetailUrl, `companyAddress`→registeredAddress |
| DOM提取 | CSS选择器 | `[class*="jobType"]`→titleCategory, `[class*="urgent"]`/文本"急"→isUrgent, 公司链接→companyDetailUrl |
| 详情页 | page.evaluate | `[class*="regAddress"]`→registeredAddress, `[class*="category"]`→titleCategory, "急聘"检测→isUrgent, 公司详情链接→companyDetailUrl |

`buildJobData` 和 `generateBasicJob` 均沿用 `basicInfo`（列表页）→ `detail`（详情页）的优先级链填充 5 个新字段。
