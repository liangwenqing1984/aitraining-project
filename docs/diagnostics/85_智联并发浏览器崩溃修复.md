# 智联并发详情页浏览器崩溃修复

**日期**：2026-04-29
**分支**：with-skills
**提交**：最新
**关联任务**：e02624e8-a508-4d54-8040-e3953182610b

---

## 一、问题现象

任务 `e02624e8`（62 关键词 × 1 城市，并发数=5）启动后 30 秒内即崩溃：

```
02:29:47  🔍 浏览器健康检查: 连接正常，当前标签页数: 2
02:29:47  ⚡ 使用并行模式处理当前批次（并发数=3）
02:29:47  [1/16] ❌ 浏览器连接已断开
02:29:47  [2/16] ❌ 浏览器连接已断开
02:29:47  [3/16] ❌ 浏览器连接已断开
02:29:56  ❌ 爬取第 1 页时出错: Attempted to use detached Frame '0C9F441F...'
02:29:56  🚨 检测到浏览器崩溃！错误: Connection closed.
```

**最终结果**：0 条数据，任务失败。

---

## 二、根因分析

浏览器在列表页解析完成后、创建详情页标签页前断开连接。3 个并发任务全部在 **1ms 内**失败，但代码存在 4 个 Bug 导致崩溃未被正确处理：

### Bug 1：浏览器断开时未标记错误类型

```typescript
// 修复前 (zhilian.ts:1205)
if (!browser.isConnected()) {
  return { success: false, data: null, index: globalIndex };
  // ❌ 没有 error 字段，下游检查匹配不到
}
```

### Bug 2：异常处理同样遗漏标记

```typescript
// 修复前 (zhilian.ts:1216)
} catch (e: any) {
  return { success: false, data: null, index: globalIndex };
  // ❌ 同上
}
```

### Bug 3：批次断开检测不完整

```typescript
// 修复前 (zhilian.ts:1352-1356)
const browserDisconnected = batchResults.some(r =>
  r.status === 'fulfilled' && r.value.error === 'BROWSER_DISCONNECTED'
  // ❌ 未检测 BROWSER_CRASHED
);
```

### Bug 4：批次全失败后继续执行

3 个全部失败后，代码进入下一批次循环，在浏览器已断开的情况下尝试创建新页面，导致 `page.evaluate()` 抛出 "Attempted to use detached Frame" 错误，该错误不在 `isBrowserCrash` 匹配列表中，进入通用错误处理最终崩溃。

---

## 三、修复方案

### 修复 1 (zhilian.ts:1205) — 标记断开错误

```typescript
if (!browser.isConnected()) {
  this.log('error', `[${globalIndex}/${filteredJobs.length}] ❌ 浏览器连接已断开`);
  return { success: false, data: null, index: globalIndex, error: 'BROWSER_DISCONNECTED' };
}
```

### 修复 2 (zhilian.ts:1216) — catch 同步标记

```typescript
} catch (e: any) {
  this.log('error', `[${globalIndex}/${filteredJobs.length}] ❌ 浏览器状态检查失败: ${e.message}`);
  return { success: false, data: null, index: globalIndex, error: 'BROWSER_DISCONNECTED' };
}
```

### 修复 3 (zhilian.ts:1352-1357) — 扩展断开检测

```typescript
const browserDisconnected = batchResults.some(r =>
  r.status === 'fulfilled' && (
    r.value.error === 'BROWSER_DISCONNECTED' ||
    r.value.error === 'BROWSER_CRASHED'
  )
);
```

### 修复 4 (zhilian.ts:1362-1366) — 全失败立即中断

```typescript
const allFailed = successCount === 0 && failCount === batch.length;
if (allFailed && !browser.isConnected()) {
  this.log('error', `💥 批次全失败且浏览器已断开，立即停止`);
  break;
}
```

### 修复 5 (zhilian.ts:1509) — 新增 detached Frame 检测

```typescript
const isBrowserCrash = error.message.includes('Connection closed') ||
                       error.message.includes('Session closed') ||
                       error.message.includes('Target closed') ||
                       error.message.includes('Protocol error') ||
                       error.message.includes('detached') ||        // ← 新增
                       error.message.includes('浏览器连接已断开') ||
                       error.message.includes('BROWSER_RESTART_SCHEDULED');
```

---

## 四、修复后的处理流程

```
并发批次开始
  → 健康检查 → isConnected()=false → 返回 BROWSER_DISCONNECTED ✓
  → 收集批次结果
  → 检测 batchDisconnected → 匹配 BROWSER_DISCONNECTED/CRASHED ✓
  → 检测 allFailed + !isConnected → break 立即跳出 ✓
  → 外层的 isBrowserCrash 检测 → 匹配 'detached' ✓
  → 触发浏览器崩溃恢复 → 重启浏览器 + 断点续传 ✓
```
