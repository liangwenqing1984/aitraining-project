# 多组合任务爬取进度显示全面优化

**日期**：2026-04-29
**分支**：with-ai
**提交范围**：bbb7338 → d21b5dd（6 次提交）

---

## 一、问题背景

多组合任务（如 25 关键词 × 1 城市 = 25 组合）的进度显示存在多个问题：

1. **进度跳跃失真**：早期组合就跳到 80%+，长时间停在 99%
2. **组合编号显示错误**：`task.current` 被记录数覆盖，如爬了 40 条记录后显示 `40/25 组合`
3. **只有单一进度条**：无法区分整体组合进度和当前组合内的记录进度
4. **组合切换时进度不归零**：第 2 组合内进度延续第 1 组合的累计记录数
5. **缺少组合内记录数统计**

---

## 二、优化内容

### 2.1 双进度条架构

**任务列表**（crawler/Index.vue）：

单进度条显示组合比例 `已完成组合数/总组合数 × 100%`，干净直观。

**监控/详情页**（TaskMonitor.vue）：

两条进度条：

| 进度条 | 含义 | 样式 |
|--------|------|------|
| 整体进度 | 已完成组合数/总组合数 | 粗 28px，蓝色，带百分比文字 |
| 当前组合内进度 | 当前组合内记录爬取进展 | 细 18px，绿色 |
| 第 X 组合标签 | 当前正在处理的组合编号 | 绿色文字徽章 |

### 2.2 多组合进度公式重设计

**修复前**（taskService.ts）：
```typescript
// 混合公式：组合基准 + 记录估算缩放 → 导致跳跃
completedBase = completedCombo / totalCombos × 100
recordEstimate = ...  // 0-99%
withinCombo = (recordEstimate / 100) × perCombo
progressPercent = completedBase + withinCombo
progressPercent = Math.max(progressPercent, comboProgress)  // 取最大值
```

**修复后**：
```typescript
// 主进度纯用组合比例，干净不跳
progressPercent = Math.min(99, Math.round(completedCombo / totalCombinations × 100))
```

### 2.3 current 字段分离

`task.current` 在系统中被双重使用，造成数值混乱：

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| 多组合任务 DB 写入 | 爬虫写组合编号 | 爬虫写组合编号（taskService 不再覆盖） |
| 多组合 WebSocket 事件 | `current = totalRecords`（记录数） | `current = comboCurrent`（组合编号） |
| 单组合任务 | 不变 | 不变 |

前端 `getComboInfo` 默认值由 `|| 1` 改为 `|| 0`，避免未完成任何组合时显示 `1/25`。

### 2.4 组合内进度归零

新增组合切换检测机制：

```typescript
// taskService.ts — taskProgress Map 追踪
lastComboIndex: 0,       // 上次组合编号
comboStartRecords: 0     // 当前组合开始时的累计记录数

// 检测组合切换 → 重置基准
if (comboCurrent > taskInfo2.lastComboIndex) {
  taskInfo2.lastComboIndex = comboCurrent;
  taskInfo2.comboStartRecords = totalRecords;
}
comboRecords = totalRecords - comboStartRecords;  // 当前组合增量
```

`comboProgress` 基于 `comboRecords`（增量）计算，切换组合时自动归零。

### 2.5 组合内记录数统计 + 跳动动画

监控页新增"当前组合采集"字段，WebSocket 推送 `comboRecords`：

```
┌─────────────────────────────────┐
│ 当前组合采集    │  已保存记录    │
│     12          │     154       │  ← 数字变动时跳动
└─────────────────────────────────┘
```

动画实现：`watch` 监听数值变化 → 添加 `.counting` class → `scale(1.25)` + 蓝色高亮 → 200ms 后自动恢复。

## 四、WebSocket 事件字段说明

`task:progress` 事件新增/变更字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `progress` | number | 主进度（多组合=组合比例，单组合=记录估算） |
| `comboProgress` | number | 当前组合内进度（0-99%），单组合=0 |
| `comboRecords` | number | 当前组合内记录增量，单组合=0 |
| `current` | number | 多组合=已完成组合编号，单组合=记录数 |
| `recordCount` | number | 总已保存记录数（不变） |
