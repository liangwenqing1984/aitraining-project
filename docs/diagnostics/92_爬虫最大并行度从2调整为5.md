# 爬虫最大并行度从 2 调整为 5

## 变更

将智联爬虫详情页抓取的并发上限从 2 调整为 5：

```typescript
// zhilian.ts:1172
const maxConcurrency = 5;          // 硬上限（原为 2 → 3 → 5）
const actualConcurrency = Math.min(concurrency, maxConcurrency);
const batchSize = actualConcurrency;
```

## 变更历史

| 时期 | maxConcurrency | 原因 |
|------|---------------|------|
| 初始 | 5 | 初始设计 |
| 第一次降低 | 3 | Chrome OOM 崩溃频繁 |
| 第二次降低 | 2 | 一页中 3 个 `page.goto()` 同时打开大型详情页仍会 OOM |
| 当前 | 5 | 增加 Chrome 内存稳定性标志后恢复 |

## 配套优化

本次恢复 5 并发的前提是已实施以下内存稳定性优化：

1. **Chrome 内存标志** (`--max-old-space-size=512` 等)：限制 V8 堆内存，防止 OOM
2. **abortableSleep**：可中断延迟，减少无效等待
3. **浏览器崩溃恢复守卫**：中止时跳过恢复流程，避免误判

## 效果

- 详情页抓取速度提升 2.5 倍（5 并发 vs 2 并发）
- WAF 检测到串行模式时会降级处理，实际并发由 `Math.min(concurrency, maxConcurrency)` 动态控制
