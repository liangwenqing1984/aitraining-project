## 经验教训

1. **避免使用通配符选择器 `[class*="*"]`**：CSS 类名是不可控的，模块化后的哈希极易产生误匹配
2. **先做精准检测再做宽泛检测**：`isSecurityVerification`（检测 title）是精准的主防线，辅助选择器只做配角
3. **日志驱动开发**：两次问题都通过日志精确还原了现场，日志的完整性直接决定调试效率
4. **分层防御**：主防线（title 检测）→ 辅助防线（特定 selectors）→ 兜底（content-length 检测），每层只处理自己能精准识别的场景
