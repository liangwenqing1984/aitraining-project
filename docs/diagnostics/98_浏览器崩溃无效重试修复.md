# 浏览器崩溃无效重试修复

## 涉及提交

| 提交 | 修复内容 |
|------|---------|
| `fea955a` | 浏览器崩溃后停止无效重试，立即触发恢复机制 |

---

## 问题

任务 `b17b989f-18a6-42a8-9bc1-27114eda8858` 浏览器崩溃后，每个待抓取的详情页仍然进行 4 次重试（每次创建新 page 失败 → 等待 → 再试），浪费大量时间后才最终触发 taskService 的浏览器重启。

```
⚠️ 抓取详情页失败 (尝试 1/4): Protocol error (Emulation.setTouchEmulationEnabled): Session closed.
⚠️ 抓取详情页失败 (尝试 2/4): Protocol error (Emulation.setTouchEmulationEnabled): Session closed.
⚠️ 抓取详情页失败 (尝试 3/4): Protocol error (Emulation.setTouchEmulationEnabled): Session closed.
⚠️ 抓取详情页失败 (尝试 4/4): Protocol error (Emulation.setTouchEmulationEnabled): Session closed.
```

浏览器已彻底断开，但每个 job 仍然跑满 4 轮重试，然后生成降级数据返回。外层 taskService 没有收到 `canRecover: true` 的错误，不会触发浏览器重启，组合中后续职位继续在"假成功"下处理。

---

## 根因

两处代码在浏览器崩溃后未及时止损：

### 1. fetchJobDetail catch 块（并发模式）

浏览器断开后，catch 块只判断重试次数是否达到 `maxRetries`，每次仍创建新 page。新 page 在已断开的浏览器上必然失败，但错误被 catch 吞噬 → 等待 → 再次重试。

### 2. 串行模式 catch 块

所有错误一律转入 `generateBasicJob` 降级兜底，包括浏览器崩溃。taskService 收到的 `{ success: false, data: {...} }` 被当作"有降级数据"正常处理，不触发浏览器重启。

---

## 修复

### 1. fetchJobDetail catch — 检测死浏览器，立即停止重试

```typescript
} catch (error: any) {
  const isBrowserGone = error.message.includes('浏览器连接已断开')
    || error.message.includes('浏览器实例已断开连接')
    || error.message.includes('Session closed')
    || error.message.includes('Connection closed')
    || error.message.includes('Target closed')
    || error.message.includes('Protocol error');

  if (isBrowserGone) {
    this.log('error', `💥 浏览器会话已断开（已重试${retryCount}次），停止重试并抛出`);
    if (page) { try { await page.close(); } catch {} }
    throw error;  // 直接抛出，触发外层浏览器恢复
  }
  retryCount++;
  // 非浏览器崩溃的正常重试逻辑...
}
```

### 2. 串行模式 catch — 浏览器崩溃时重新抛出

```typescript
} catch (error: any) {
  const isBrowserCrash = error.message.includes('Connection closed')
    || error.message.includes('Session closed')
    || error.message.includes('Target closed')
    || error.message.includes('Protocol error')
    || error.message.includes('Browser closed');

  if (isBrowserCrash) {
    throw error;  // 不保存降级数据，直接触发浏览器重启后重试本 job
  }
  jobData = this.generateBasicJob(job, config);  // 非崩溃的正常降级
}
```

## 效果

|------|--------|--------|
