# 智能数据分析模块 - 快速测试指南

## 🚀 快速开始

### 1. 启动服务

```bash
# 方式一：使用项目脚本（推荐）
.\start-dev.bat

# 方式二：手动启动
# 终端1 - 后端
cd code/backend
npm run dev

# 终端2 - 前端  
cd code/frontend
npm run dev
```

### 2. 访问系统

打开浏览器访问：`http://localhost:5173`

## 📝 测试流程

### Step 1: 创建爬取任务

1. 点击左侧菜单 **"爬虫任务"**
2. 点击 **"新建任务"** 按钮
3. 填写任务配置：
   - **任务名称**：Java开发工程师-北京
   - **数据来源**：智联招聘 / 前程无忧
   - **职位关键词**：输入 `Java` 后按回车或点击"添加"
   - **城市**：输入 `北京` 后按回车或点击"添加"
   - **最大页数**：5（测试用，不用太多）
4. 点击 **"启动任务"**

### Step 2: 等待任务完成

1. 在任务列表中观察任务状态
2. 状态变化：`待执行` → `爬取中` → `已完成`
3. 查看实时日志了解爬取进度
4. 等待状态变为 **"已完成"**

### Step 3: 查看数据分析

#### 方式一：从文件管理进入
1. 点击左侧菜单 **"文件管理"**
2. 找到刚生成的CSV文件
3. 点击操作列的 **"分析"** 按钮
4. 自动跳转到数据分析页面

#### 方式二：直接访问（如果知道fileId）
```
http://localhost:5173/analysis?fileId=YOUR_FILE_ID
```

## 🐛 常见问题排查

### 问题1：分析页面空白

**可能原因：**
- CSV文件为空
- 后端分析接口报错

**排查步骤：**
1. 打开浏览器F12开发者工具
2. 查看Console标签是否有错误
3. 查看Network标签，找到`/files/{id}/analyze`请求
4. 检查响应状态码和数据

**解决方案：**
- 确保任务已成功完成并生成了CSV文件
- 检查后端日志：`code/backend`目录下的终端输出

### 问题2：图表不显示

**可能原因：**
- 对应字段无数据
- ECharts未正确初始化

**排查步骤：**
1. 检查Console是否有ECharts相关错误
2. 查看分析结果数据结构：
   ```javascript
   console.log(analysisResult.value)
   ```
3. 检查对应computed属性是否有数据：
   ```javascript
   console.log(salaryDistributionData.value)
   ```

**解决方案：**
- 某些图表无数据是正常的（如CSV中没有薪资字段）
- 确保已安装echarts依赖：`npm list echarts`

### 问题3：薪资数据显示为0

**可能原因：**
- CSV中薪资字段格式不规范
- 薪资解析正则表达式未匹配

**排查步骤：**
1. 下载CSV文件用Excel打开
2. 检查"薪资范围"列的内容格式
3. 支持的格式示例：
   - `10K-15K`
   - `10000-15000`
   - `10-15K`
   - `1万-1.5万`

**解决方案：**
- 这是正常的，不是所有招聘网站都提供薪资格式化数据
- 可以手动修改后端解析逻辑适配更多格式

### 问题4：智能洞察数量为0

**可能原因：**
- 数据量太少
- 关键字段缺失

**排查步骤：**
1. 检查`analysisResult.insights`数组长度
2. 确认是否有以下字段：
   - 薪资字段（包含"薪资"）
   - 城市字段（包含"城市"）
   - 经验字段（包含"经验"）
   - 学历字段（包含"学历"）

**解决方案：**
- 增加爬取数据量（增大maxPages参数）
- 确保爬取的网站提供这些字段信息
