## 问题

任务 `6f6731cc-8536-4190-ac61-b5c01b9a0c6c` 使用 `concurrency=1`（串行模式），仅处理 2 个职位就报 4 次警告：

```
⚠️ 抓取详情页失败 (尝试 1/4): 未能提取到职位标题或公司名称，页面可能加载不完整
```

第二个职位连续 3 次重试均失败，直到用户中止任务。

---

## 根因

串行模式调用旧版 `fetchJobDetail`，其等待和提取逻辑与新版 `fetchJobDetailWithPage`（并发模式使用）存在显著差距：

### 1. 等待机制不足

旧版导航到详情页后仅 `sleep(2000)` 固定等待 2 秒。日志显示每次失败都在导航后 **2.3 秒**，恰好是 2 秒等待 + 少量执行时间。部分页面的职位标题需要更长时间才能 JS 渲染完毕。

新版使用 `waitForSelector` 主动等待关键元素出现（6s 超时）+ 额外 4s 缓冲 = 最多 10s。

### 2. 仅单个标题选择器

旧版只查找 `.summary-planes__title`，新版有 5 个 fallback 选择器覆盖不同 DOM 版本。

### 3. 未提取公司名（严重 bug）

旧版 `page.evaluate` 中 **从未设置 `result.company`**。代码：

```typescript
// 旧版：只解析 companyNature/companyScale/businessScope，没有 result.company
const companyDescEl = document.querySelector('.company-info__desc');
if (companyDescEl) {
  const companyText = (companyDescEl.textContent || '').trim();
  const parts = companyText.split('·').map(p => p.trim());
  result.companyNature = parts[0];  // ← 有
  result.companyScale = parts[1];   // ← 有
  // result.company 从未设置！       // ← 缺失
}
```

验证条件 `if (!detail.title && !detail.company)` 中 `detail.company` 永远为 `undefined`，导致仅凭标题判断——标题为空就判定失败，即使公司信息可能已获取。

### 4. 无页面健康检查

旧版不检测 bodyLength、不做内容过少刷新重试。

---

## 修复 (`xxxxxxx`)

将旧版 `fetchJobDetail` 的等待和提取逻辑完整对齐新版 `fetchJobDetailWithPage`：

### 等待机制升级

```typescript
// 修复前
await this.abortableSleep(2000);  // 固定2秒

// 修复后
await page.waitForSelector('.summary-planes__title, .company-name, [class*="job-title"]', {
  timeout: 6000  // 主动等待最多6秒
});
await this.abortableSleep(2000);  // 额外缓冲
```

### 提取逻辑升级

- **标题**：1 个选择器 → 5 个 fallback 选择器
- **公司名**：新增 5 个选择器提取 `result.company`
- **所有字段**：添加 `[class*="..."]` 模糊匹配 fallback

### 新增页面健康检查

- WAF "Security Verification" 检测
- bodyLength < 1000 → reload 重试
- 刷新后再次检查，仍失败则明确抛 `PAGE_LOAD_FAILED`

### 新增诊断日志

```typescript
this.log('info', `📊 提取结果: 标题="${detail.title}", 公司="${detail.company}"`);
```
