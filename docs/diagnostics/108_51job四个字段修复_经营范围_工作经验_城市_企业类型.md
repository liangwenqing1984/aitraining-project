## 涉及提交

| 提交 | 说明 |
|------|------|

## 问题

| 问题 | 涉及字段 |
|------|---------|
| 字段为空 | 经营范围 |
| 内容不正确 | 工作经验（全句而非简洁形式）、城市（含区/县）、企业类型（匹配到不相关内容） |
| 多余列 | 职称分类（用户不需要） |

## 根因分析

### 1. 经营范围为空

详情页为旧版 `jobs.51job.com` 格式，经营范围有两种出现形式：

**形式 A：子章节标题**（如华康世纪医疗）
```
（二）经营范围
1、医疗专项集成化整体解决方案：...
2、实验室集成化整体解决方案：...
```

`findByLabel(['经营范围', ...])` 能找到"经营范围"所在行，但标签位于行末且无同行的值，跨行逻辑只取下一行，丢掉了后续多行内容。

**形式 B：嵌入公司描述**（大多数公司）
```
公司信息 北京怡成生物电子技术股份有限公司，公司规模为50-150人，公司类型为民营公司，所属行业为其他专业服务。
```

无独立的"经营范围"标签，信息嵌入公司描述段落中。

### 2. 工作经验全句捕获

CSS 元素遍历中存在两个问题：

- **`t.indexOf('经验') !== -1` 过于宽泛**：`"7年以上医疗净化工程...相关销售工作经验"` 包含"经验"二字，整句被赋值
- **`\d+年以上` 不匹配 `\d+年及以上`**：`"8年及以上"` 无法被简洁形式正则匹配

### 3. 城市含区/县

51job 旧版页面城市格式为 `北京-海淀区`、`北京-朝阳区`。现有清理逻辑只处理 `北京市朝阳区` 形式（`replace(/[市].+$/, '市')`），对 `-区` 格式无效。

### 4. 企业类型不正确

- CSS 选择器 `[class*="type"]` 过于宽泛，可能匹配到职位类型标签（如"全职"、"销售总监"）
- 正则兜底缺少"已上市"、"创业公司"等模式
- `findByLabel` 可能匹配到公司描述中的长句而非简洁类型词
- 未做标准化：`已上市` 应映射为 `上市公司`，`民营公司` 应映射为 `民营企业`

### 5. 职称分类多余

`titleCategory`（职称分类）在 Excel 中始终为空或为错误值（CSS 选择器匹配到不相关内容后长度超 15 被清空），用户不需要此列。

## 修复方案

### 经营范围：三层提取

```
1. findByLabel（标签匹配，已有）
   └─ 失败 → 继续

2. 多行提取（新增）
   ├─ 遍历文本行找包含"经营范围"的行
   ├─ 取后续最多 8 行，遇到章节标题或短行停止
   └─ 失败 → 继续

3. 公司描述兜底（新增）
   └─ 正则提取"公司信息"后 20-300 字
```

```javascript
// 多行提取
if (!result.businessScope || result.businessScope.length < 10) {
  for (var li2 = 0; li2 < textLines.length; li2++) {
    if (textLines[li2].indexOf('经营范围') !== -1) {
      var scopeLines = [];
      for (var si = li2 + 1; si < Math.min(textLines.length, li2 + 8); si++) {
        var sl = textLines[si];
        if (sl.length < 4 || sl.indexOf('（') === 0 || sl.indexOf('公司信息') !== -1) break;
        scopeLines.push(sl);
      }
      if (scopeLines.length > 0) { result.businessScope = scopeLines.join('；'); break; }
    }
  }
}
// 公司描述兜底
if (!result.businessScope) {
  var companyDescMatch = bodyText.match(/公司信息[\s\S]{20,300}?(?=登录|更多详情|$)/);
  if (companyDescMatch) {
    var descText = companyDescMatch[0].replace(/公司信息/, '').trim();
    if (descText.length > 15) result.businessScope = descText.substring(0, 200);
  }
}
```

### 工作经验：提取简洁形式

```javascript
// 修复前
if (/\d+-?\d*年/.test(t) || t.indexOf('经验') !== -1)
  result.experience = result.experience || t;  // 直接赋全文本

// 修复后
if (!result.experience && (/\d+-?\d*年/.test(t) || t.indexOf('经验') !== -1)) {
  var conciseExp = t.match(
    /(\d+-\d+年|\d+年以上|\d+年及以上|\d+年经验|经验不限|应届生|无需经验|在校生|\d+年)/
  );
  result.experience = conciseExp ? conciseExp[0] : t.substring(0, 30);
}
```

关键改动：
- 正则增加 `\d+年及以上` 模式
- 从匹配文本中**提取**简洁形式（capture group），而非使用原始文本
- 匹配失败时截取前 30 字符兜底（避免长句）

### 城市格式：三步清理 + 直辖市补全

```javascript
// 修复前
var cityClean = rawCity.replace(/[市].+$/, '市');  // 只处理 "北京市朝阳区"

// 修复后
var cityClean = rawCity
  .replace(/[市].+$/, '市')      // "北京市朝阳区" → "北京市"
  .replace(/-[^,\s]+$/, '')      // "北京-海淀区" → "北京"
  .replace(/[市]$/, '');         // "北京市" → "北京"
// ...
// 直辖市统一补全"市"后缀
var baseCity = result.city.replace(/[市]$/, '');
if (/^(北京|上海|天津|重庆)$/.test(baseCity)) {
  result.city = baseCity + '市';
}
```

清理流程：`北京-海淀区` → 步骤1不变 → 步骤2剥离 `-海淀区` → `北京` → 直辖市补全 → `北京市`

### 企业类型：宽松选择器移除 → 验证 → 标签匹配 → 标准化

```
1. CSS get（收窄选择器，移除 [class*="type"]）
   └─ 校验：不包含企业类型关键词 → 丢弃
   └─ 失败 → 继续

2. findByLabel（扩展正则匹配模式）
   └─ 失败 → 继续

3. 标准化映射
   ├─ 已上市/上市公司/创业板上市 → 上市公司
   ├─ 民营/有限责任 → 民营企业
   ├─ 国企/国有企业 → 国有企业
   ├─ 外商独资/外资 → 外资企业
   ├─ 合资/中外合资 → 合资企业
   ├─ 创业/初创 → 创业公司
   └─ ...（共 10 种映射）
```

### 移除职称分类

从 `constants.ts` 三处移除：
- `CSV_FIELDS_51JOB`：移除 `'职称分类'`
- `JOB51_EXCEL_KEYS`：移除 `'titleCategory'`
- `CSV_FIELD_MAP_51JOB`：移除 `titleCategory: '职称分类'`

内部数据结构保留 `titleCategory` 字段（不影响编译），仅不再输出到 Excel。
