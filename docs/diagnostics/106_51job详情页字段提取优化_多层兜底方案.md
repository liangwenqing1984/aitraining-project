# 51job 详情页字段提取优化 — 多层兜底方案

## 涉及提交

| 提交 | 说明 |
|------|------|
| `ea64d89` | 优化详情页字段提取：文本兜底 + API 映射扩展 + 城市清理 |

## 问题

用户反馈详情页抓取存在以下问题：

| 问题 | 涉及字段 |
|------|---------|
| 字段为空 | 经营范围、公司规模、注册地址、工作地址、学历 |
| 内容不正确 | 职称分类、职能类别、企业类型 |
| 格式问题 | 城市显示包含区/县（如"北京市朝阳区"），期望只显示"北京" |

## 根因分析

### 1. CSS 选择器与新版站点不匹配

详情页提取代码中的 CSS 选择器基于**旧版** `jobs.51job.com` 设计（`.cn`、`.tHeader`、`.tCompany_sidebar`、`.bmsg` 等）。51job 已迁移到 Vue.js SPA 新站点 `we.51job.com`，DOM 结构完全不同。

### 2. XHR API 字段映射不完整

搜索页 XHR 拦截的 API JSON 数据包含更多字段，但映射代码仅覆盖了基础字段（title、company、salary、city），未映射 education、experience、address、workType、jobDescription、jobTags 等。

### 3. buildJobData 缺少 basicInfo 兜底

`buildJobData` 中 `workExperience`、`education`、`workAddress` 字段仅从 detail（详情页抓取）获取，未回退到 basicInfo（XHR API 数据），导致 CSS 选择器失效时直接为空。

## 修复方案：4 层数据源逐级兜底

```
┌──────────────────────────────────────────┐
│ 1. CSS 选择器提取（保留原有，适配新旧版）   │
│    └─ 失败 → 继续                        │
├──────────────────────────────────────────┤
│ 2. 全文本提取（解析 bodyText）             │
│    ├─ 按中文标签查找（"公司规模："等）       │
│    └─ 正则模式匹配（学历/经验/福利等）       │
│    └─ 失败 → 继续                        │
├──────────────────────────────────────────┤
│ 3. XHR API 数据（扩展字段映射）             │
│    └─ 失败 → 继续                        │
├──────────────────────────────────────────┤
│ 4. basicInfo 兜底（generateBasicJob）     │
└──────────────────────────────────────────┘
```

### 第 1 层：CSS 选择器提取（保留）

原有选择器列表保持不变，增加新站点可能使用的 class：

```javascript
// 经验/学历/工作类型 — 扩大选择器范围
var infoEls = document.querySelectorAll(
  '.msg.ltype span, .jtag span, .bmsg, [class*="info"] span, [class*="require"] span, ' +
  '.t1 span, [class*="detail"] span, [class*="condition"] span, [class*="tag"] span, ' +
  '[class*="item"] span, .job-header span'  // ← 新增
);
```

### 第 2 层：全文本提取（新增核心改进）

新增 `findByLabel()` 函数，从页面全文本中按中文标签键值对查找：

```javascript
// 获取页面全文本并按行分割
var bodyText = document.body.innerText;
var textLines = bodyText.split('\n').map(function(l) { return l.trim(); });

// 按标签查找值（"公司性质：民营" → "民营"）
var findByLabel = function(labels, fallbackPattern) {
  for (var li = 0; li < textLines.length; li++) {
    var line = textLines[li];
    for (var lj = 0; lj < labels.length; lj++) {
      var idx = line.indexOf(labels[lj]);
      if (idx !== -1) {
        var val = line.substring(idx + labels[lj].length)
          .replace(/^[：:\s]+/, '').trim();
        if (val && val.length < 80) return val;
        // 值可能跨行（标签在上一行末尾，值在下一行）
        if (!val && li + 1 < textLines.length) {
          var nextLine = textLines[li + 1];
          if (nextLine.length < 80 && nextLine.indexOf('：') === -1) {
            return nextLine;
          }
        }
      }
    }
  }
  // 兜底：正则匹配
  if (fallbackPattern) {
    var m = bodyText.match(fallbackPattern);
    if (m) return m[1] || m[0] || '';
  }
  return '';
};
```

#### 各字段提取策略

| 字段 | CSS 选择器 | 文本兜底方式 |
|------|-----------|------------|
| education | `[class*="info"] span` 等 | 正则 `/(本科\|硕士\|博士\|大专\|中专\|高中\|初中\|MBA\|EMBA\|学历不限)/` |
| experience | 同上 | 正则 `/(\d+-\d+年\|\d+年以上\|\d+年经验\|经验不限\|应届生\|无需经验\|在校生)/` |
| workType | 同上 | 正则 `/(全职\|兼职\|实习\|合同制\|劳务派遣\|临时工)/` |
| recruitmentCount | 同上 | 正则 `/招(\d+)人/` |
| address | `[class*="address"]` 等 | `findByLabel(['工作地址', '上班地址', '公司地址', '办公地址', '工作地点'])` |
| businessScope | `[class*="business"]` 等 | `findByLabel(['经营范围', '主营业务', '业务范围'])` |
| registeredAddress | `[class*="regAddress"]` 等 | `findByLabel(['注册地址', '注册地', '登记地址'])` |
| companyNature | `[class*="type"]` 等 | `findByLabel(['公司性质', '企业性质', '单位性质', '企业类型', '公司类型'])` |
| companyScale | `[class*="scale"]` 等 | `findByLabel(['公司规模', '企业规模', '员工人数', '规模'])` |
| titleCategory | `[class*="jobType"]` 等 | `findByLabel(['职称分类', '职位类别', '岗位类别', '职位类型'])` |
| jobCategory | `[class*="function"]` 等 | `findByLabel(['职能类别', '职能', '工作职能', '岗位职能'])` |
| jobDescription | `[class*="description"]` 等 | 取页面最长 5 行文本拼接 |
| jobTags | `[class*="tag"]` 等 | 正则匹配 20+ 常见福利关键词 |

#### 不正确字段处理

职称分类/职能类别/企业类型之前"不正确"是因为 CSS 选择器匹配到了无关元素（如 `[class*="type"]` 匹配到页面导航中的 class）。现在文本兜底会精确查找标签行，且过滤掉过长文本：

```javascript
// 避免匹配到完整职位名：职称分类通常是较短分类词
if (result.titleCategory && result.titleCategory.length > 15) {
  result.titleCategory = '';
}
if (result.jobCategory && result.jobCategory.length > 15) {
  result.jobCategory = '';
}
```

#### 城市清理

```javascript
var rawCity = get(...);
// 去掉"市"后面的区/县/街道："北京市朝阳区" → "北京市"
var cityClean = rawCity.replace(/[市].+$/, '市');
if (cityClean.length >= 3) {
  result.city = cityClean;
} else {
  // 匹配已知城市名（处理不含"市"的简称）
  var cityMatch = rawCity.match(/(北京|上海|广州|深圳|杭州|...)/);
  result.city = cityMatch ? cityMatch[0] : rawCity;
}
```

### 第 3 层：XHR API 字段映射扩展

新增 8 个字段的 API 映射，每个字段支持多种命名变体：

```typescript
// 新增映射
education: item.education || item.degree || item.degreeName || item.eduLevel || ...
experience: item.workYear || item.workExperience || item.experience || item.workingYears || ...
workType: item.workType || item.employType || item.employmentType || item.jobNature || ...
address: item.workAddress || item.companyAddress || item.address || item.workPlace || ...
recruitmentCount: item.recruitmentCount || item.recruitCount || item.hireCount || ...
jobDescription: item.jobDescription || item.description || item.jobDesc || item.duty || ...
jobTags: item.jobTags || item.tags || item.welfare || item.benefits || ...
jobCategory: item.jobCategory || item.funcCategory || item.functionName || ...
```

同时添加 API 响应结构诊断日志：

```typescript
this.log('info', `🔍 API响应字段: ${Object.keys(items[0]).join(', ')}`);
```

### 第 4 层：basicInfo 兜底

`buildJobData` 和 `generateBasicJob` 中所有字段都添加 basicInfo 回退：

```typescript
// 修复前
workExperience: detail.experience || '',  // detail 为空 → 字段为空
education: detail.education || '',

// 修复后
workExperience: detail.experience || basicInfo.experience || '',
education: detail.education || basicInfo.education || '',
// ... 所有字段均有 basicInfo 兜底
```
