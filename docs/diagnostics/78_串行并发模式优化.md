**Commit**: `d876a9d`

## 修复内容

### 1. 变量遮蔽 Bug

**问题**: 并发模式中 `concurrency` 变量被重复声明，导致外层变量被内层覆盖：

```typescript
const concurrency = config.concurrency || 1;      // 外层声明
// ...
const concurrency = Math.min(config.concurrency || 2, maxConcurrency);  // 内层重新声明！
```

**修复**: 内层使用不同变量名 `actualConcurrency`：

```diff
- const concurrency = Math.min(config.concurrency || 2, maxConcurrency);
- const batchSize = concurrency;
+ const actualConcurrency = Math.min(concurrency, maxConcurrency);
+ const batchSize = actualConcurrency;
```

### 2. 默认并发数调整

```diff
- const concurrency = config.concurrency || 1;  // 默认串行
+ const concurrency = config.concurrency != null ? config.concurrency : 2;  // 默认并发2
```

`!= null` 判断确保 `concurrency=0` 时不会被错误地替换为默认值。

### 3. 串行模式增加随机延迟

```typescript
// 每个职位间 2-4 秒随机延迟，模拟人类浏览
if (i < filteredJobs.length - 1) {
  const delay = 2000 + Math.random() * 2000;
  await new Promise(resolve => setTimeout(resolve, delay));
}
```
