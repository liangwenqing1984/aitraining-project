## 📊 技术细节

### **为什么需要导入db？**

#### **模块系统设计：**

Node.js使用ES模块或CommonJS模块系统，每个文件是独立的模块作用域。

**当前架构：**
```
src/
├── config/
│   └── database.ts  → 导出 db 对象
├── app.ts           → 导出 io 对象
└── services/
    └── crawler/
        └── zhilian.ts  → 需要导入 db 和 io
```

**导入规则：**
- 要使用其他模块导出的内容，必须显式导入
- 不能依赖全局变量（除非明确声明为global）

**优势：**
- ✅ 编译时检查类型错误
- ✅ IDE智能提示和自动补全

**局限：**
