# 前端日志按任务 ID 独立隔离与完整下载

## 问题

### 问题 1：日志混乱

多个任务同时运行时，所有日志混在同一个全局数组中。切换到不同任务查看时，看到的是其他任务的日志，无法辨别每条日志属于哪个任务。

### 问题 2：前端日志不完整

前端仅缓存 WebSocket 推送的实时日志，页面刷新后日志全部丢失，且不会超过 500 条上限。

## 解决方案

### 1. 日志按任务 ID 隔离存储

将全局日志数组改为 `Map<taskId, logArray>`：

```typescript
// 旧代码：全局共享数组
const logs = ref<Array<...>>([])

// 新代码：按 taskId 隔离
const taskLogs = ref<Map<string, Array<...>>>(new Map())

// 兼容旧代码的计算属性
const logs = computed(() => {
  if (!currentTask.value?.id) return []
  let taskLogList = taskLogs.value.get(currentTask.value.id)
  if (!taskLogList) {
    taskLogList = []
    taskLogs.value.set(currentTask.value.id, taskLogList)
  }
  return taskLogList
})
```

### 2. 日志添加路由 — addLogToTask

```typescript
function addLogToTask(taskId: string, level: string, message: string, customTime?: string) {
  if (!taskLogs.value.has(taskId)) {
    taskLogs.value.set(taskId, [])
  }
  
  const taskLogList = taskLogs.value.get(taskId)!
  taskLogList.push({
    time: customTime || new Date().toLocaleTimeString('zh-CN', { hour12: false }),
    level,
    message
  })
  
  // 限制每个任务的日志数量（最多 500 条）
  if (taskLogList.length > 500) {
    taskLogList.shift()
  }
}
```

### 3. WebSocket 事件路由到正确任务

```typescript
// task:log 事件 — 根据 taskId 路由
socket.value.on('task:log', (data: any) => {
  if (data.taskId) {
    addLogToTask(data.taskId, data.level || 'info', data.message)
  } else {
    // 兼容旧格式
    if (currentTask.value?.id) {
      addLogToTask(currentTask.value.id, data.level || 'info', data.message)
    }
  }
})

// task:progress / task:status 等其他事件同样只添加到匹配的当前任务
```

### 4. 完整日志下载（从后端 API 读取）

```typescript
async function downloadLogs(taskId?: string) {
  const targetTaskId = taskId || currentTask.value?.id
  if (!targetTaskId) return

  // 从后端 API 获取完整日志（最多 10000 条）
  const res = await taskApi.getTaskLogs(targetTaskId, 10000)
  
  if (!res.data?.logs?.length) return

  // 格式化为文本
  const logContent = res.data.logs.map(log => {
    const timestamp = log.timestamp || new Date().toISOString()
    const level = log.level ? `[${log.level.toUpperCase()}]` : '[INFO]'
    return `${timestamp} ${level} ${log.message}`
  }).join('\n')

  // 触发浏览器下载
  const blob = new Blob([logContent], { type: 'text/plain;charset=utf-8' })
  const url = URL.createObjectURL(blob)
  const link = document.createElement('a')
  link.href = url
  link.download = `task_${targetTaskId}_${timestamp}.log`
  document.body.appendChild(link)
  link.click()
  document.body.removeChild(link)
  URL.revokeObjectURL(url)
}
```

### 5. 任务切换与清理

```typescript
// 切换任务时初始化日志数组
function setCurrentTask(task: Task | null) {
  currentTask.value = task
  if (task?.id && !taskLogs.value.has(task.id)) {
    taskLogs.value.set(task.id, [])
  }
}

// 删除任务时清理日志
taskLogs.value.delete(taskId)

// 清空指定任务日志
function clearLogs(taskId?: string) {
  const targetTaskId = taskId || currentTask.value?.id
  if (targetTaskId && taskLogs.value.has(targetTaskId)) {
    taskLogs.value.set(targetTaskId, [])
  }
}
```

## 效果

- 切换任务时日志完全隔离，互不干扰
- 每个任务最多缓存 500 条实时日志（内存安全）
- 下载功能从后端 API 读取完整日志文件，不受前端缓存限制
- 下载文件名使用完整任务 ID 命名，便于识别
