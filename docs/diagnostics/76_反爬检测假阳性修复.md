## 问题描述

WAF 增强方案部署后，所有详情页被误判为反爬页面，100% 假阳性。

### 日志特征

```
📊 页面健康检查: body长度=756849, 有标题=true, 有公司=true   ← 页面实际加载成功！
🚨 检测到反爬拦截！
页面标题="鸿蒙智行汽车销售顾问（尊界）招聘_龙海汽车销售集团股份有限公司招聘 - 智联招聘"
❌ ANTI_BOT_DETECTED: 检测到反爬拦截（验证码/错误页面）
```

### 关键线索

| 指标 | 实际加载成功的页面 | Security Verification 页面 |
|------|---------------------|----------------------------|
| body 大小 | **750KB+** | 846 字节 |
| hasTitle | **true** | false |
| hasCompany | **true** | false |
| 页面标题 | 正常职位标题 | "Security Verification" |

body 750KB+ 且 hasTitle=true 的页面被误判为反爬页面，说明选择器有假阳性。

---

## 根因

`fetchJobDetailWithPage` 中的 `hasErrorPage` 检测器使用了过于宽泛的 CSS 选择器：

```javascript
hasErrorPage: !!document.querySelector(
  '.error-page, .captcha, #verifyCode, [class*="verify"], [class*="captcha"], [class*="robot"]'
)
```

**`[class*="verify"]`** 匹配了智联招聘详情页 HTML 中所有包含 "verify" 字符串的类名。现代前端框架（如 Vue/React）会生成哈希类名，其中可能包含 "verify" 子串，导致正常页面被误判。

类似地，`hasLoginPrompt` 中的 `[class*="login"]` 和 `[class*="登录"]` 也可能匹配到正常的导航元素。

---

## 修复

移除所有 `[class*="*"]` 通配符选择器，改为精确匹配：

```diff
- hasErrorPage: !!document.querySelector(
-   '.error-page, .captcha, #verifyCode, [class*="verify"], [class*="captcha"], [class*="robot"]'
- ),
+ hasErrorPage: !!document.querySelector(
+   '.error-page, #verifyCode, .robot-check, [class*="robot"]'
+ ),

- hasLoginPrompt: !!document.querySelector(
-   '[class*="login"], [class*="登录"], .need-login'
- ),
+ hasLoginPrompt: !!document.querySelector(
+   '.need-login, .login-prompt, [data-require="login"]'
+ ),
```

### 设计原则

- **主防线由 `isSecurityVerification` 承担**：直接检测 `document.title`，精准可靠
- **辅助检测器只做精确匹配**：不用 `[class*="*"]` 通配，只用具体 class/id/属性名
- **宁可漏过不可误杀**：WAF 真实拦截（body<1000 + title="Security Verification"）是主要检测目标
