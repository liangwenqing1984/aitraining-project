# 爬取日志持久化功能 - 快速测试指南

## 🚀 快速开始

### 1. 启动服务

```bash
# 终端1: 启动后端
cd code/backend
npm run dev

# 终端2: 启动前端
cd code/frontend
npm run dev
```

### 2. 创建测试任务

1. 访问 `http://localhost:3000/crawler`
2. 点击"创建任务"按钮
3. 配置任务参数:
   - **数据来源**: 选择"智联招聘"或"前程无忧"
   - **职位关键词**: 输入 `Java工程师` (点击"添加")
   - **城市**: 选择 `北京`
   - **最大页数**: 设置为 `2` (快速测试)
4. 点击"保存并启动"

### 3. 观察实时日志

任务启动后,在"爬取详情"页面应该看到:

```
[10:30:01] [TaskService] 开始启动任务: abc123-def456
[10:30:01] [ConsoleInterceptor] 📝 日志文件已创建: .../task_abc123.log
[10:30:02] [ZhilianCrawler] 开始爬取: 关键词="Java工程师" | 城市="北京"
[10:30:03] [ZhilianCrawler] 📄 正在请求第 1 页: https://www.zhaopin.com/...
[10:30:05] [ZhilianCrawler] ✅ 第 1 页处理完成 | 耗时 2.34秒 | 解析 30 条
...
```

**验证点:**
- ✅ 日志实时滚动更新
- ✅ 不同级别日志颜色不同(INFO=绿色, WARN=黄色, ERROR=红色)
- ✅ 自动滚动到最新日志

### 4. 验证日志文件生成

打开新终端,执行:

```bash
# 查看日志目录
ls -lh code/backend/data/logs/

# 应该看到类似输出:
# -rw-r--r--  1 user  staff   15K Apr 23 10:30 task_abc123-def456.log

# 实时查看日志文件内容
tail -f code/backend/data/logs/task_*.log
```

**验证点:**
- ✅ 日志文件已创建
- ✅ 内容与前端显示一致
- ✅ 包含时间戳和日志级别

### 5. 等待任务完成

任务完成后(状态变为"已完成"),继续下一步测试。

### 6. 测试历史日志加载

**关键测试步骤:**

1. **刷新浏览器页面** (按 F5)
2. **重新进入该任务的详情页**:
   - 返回任务列表
   - 点击刚才的任务
   - 进入"爬取详情"页面

3. **观察"实时日志"区域**:
   - ✅ 应该立即显示所有历史日志(不需要等待)
   - ✅ 日志数量与任务运行时的总数一致
   - ✅ 时间戳正确显示(如 `10:30:01`)
   - ✅ 日志顺序正确(按时间从早到晚)
   - ✅ 自动滚动到底部,显示最新日志

4. **检查浏览器控制台**:
   ```
   [TaskMonitor] 📋 开始加载历史日志...
   [TaskMonitor] ✅ 成功加载 152 条历史日志
   ```

### 7. 测试 API 接口

```bash
# 替换 {taskId} 为实际的任务ID
curl http://localhost:3004/api/tasks/{taskId}/logs?limit=10 | jq .

# 应该看到类似响应:
{
  "success": true,
  "data": {
    "taskId": "abc123-def456",
    "logs": [
      {
        "timestamp": "2026-04-23T10:30:01.123Z",
        "level": "info",
        "message": "[TaskService] 开始启动任务: abc123-def456"
      },
      ...
    ],
    "totalLines": 152,
    "hasMore": false
  }
}
```

**验证点:**
- ✅ API 返回成功
- ✅ logs 数组不为空
- ✅ 每条日志包含 timestamp, level, message
- ✅ totalLines 显示总行数
- ✅ hasMore 指示是否有更多日志

### 8. 测试日志连续性(断点续传场景)

1. 启动一个长时间运行的任务(设置 maxPages=10)
2. 等待爬取几页后,手动停止任务
3. 查看日志文件,应该有:
   ```
   ================================================================================
   结束时间: 2026-04-23T10:35:00.000Z
   ================================================================================
   ```
4. 恢复任务
5. 查看日志文件:
   ```
   ================================================================================
   任务ID: abc123-def456
   开始时间: 2026-04-23T10:36:00.000Z
   ================================================================================
   
   [2026-04-23T10:36:01.123Z] [INFO] [TaskService] 🔄 断点续传模式...
   ...
   ```

**验证点:**
- ✅ 日志文件中有多次"开始时间"和"结束时间"标记
- ✅ 日志连续追加,没有丢失
- ✅ 前端显示完整的日志历史

## 🐛 常见问题排查

### 问题1: 日志文件未生成

**检查步骤:**
```bash
# 1. 确认目录存在
ls -la code/backend/data/

# 2. 如果没有 logs 目录,手动创建
mkdir -p code/backend/data/logs

# 3. 检查后端控制台是否有错误
# 查找 "[ConsoleInterceptor]" 相关日志
```

**可能原因:**
- 目录不存在
- 权限不足
- 任务未真正启动(ConsoleInterceptor 未初始化)

### 问题2: 前端看不到历史日志

**检查步骤:**
```bash
# 1. 打开浏览器开发者工具 → Network 标签
# 2. 刷新页面,查找请求: /api/tasks/{taskId}/logs
# 3. 检查响应状态码和内容

# 4. 如果请求失败,检查后端控制台
# 查找 "[TaskController] 📋 获取任务日志" 相关日志
```

**可能原因:**
- 后端路由未注册(检查 `taskRoutes.ts`)
- 日志文件不存在
- API 调用失败(网络错误或跨域问题)

### 问题3: 日志时间戳显示异常

**检查步骤:**
```javascript
// 在浏览器控制台执行
console.log(new Date().toLocaleTimeString('zh-CN', { hour12: false }))
// 应该显示当前时间,如 "10:30:01"

// 检查日志文件中的原始时间戳
cat code/backend/data/logs/task_*.log | head -20
// 应该看到 ISO 格式: [2026-04-23T10:30:01.123Z]
```

**可能原因:**
- 时区设置不正确
- 时间戳格式解析失败
- 前端格式化逻辑有误

### 问题4: 日志加载缓慢

**症状**: 刷新页面后,需要等待几秒才看到日志

**优化建议:**
```typescript
// 在 taskController.ts 中减少返回数量
const { limit = 500 } = req.query;  // 从 1000 改为 500

// 或在 TaskMonitor.vue 中传入 limit 参数
const res = await taskApi.getTaskLogs(taskId, 200)  // 只加载最近200条
```
