# 项目文件清理

**日期**: 2026-04-27  
**分支**: `with-deepseek`  
**Commit**: `1357911`

---

## 背景

项目根目录和 `code/backend/` 下积累了大量临时文件，影响项目可维护性。

---

## 清理内容

### 1. 归档诊断报告（65 个文件）

根目录 `.md` 诊断报告按生成时间编号，移入 `docs/diagnostics/`：

```
01_SERIAL_DELAY_OPTIMIZATION.md  (最新 2026-04-27)
   ...
65_OAUTH2_FINAL_VERIFICATION.md  (最早 2026-04-21)
```

### 2. 删除临时文件

| 类别 | 数量 | 位置 |
|------|------|------|
| 临时分析脚本 | 30+ | `code/backend/analyze-*.js`, `diagnose-*.js`, `check-*.js` 等 |
| 测试脚本 | 20+ | `code/backend/test-*.js` |
| xlsx 数据文件 | 80+ | `code/backend/data/csv/*.xlsx` |
| 源码备份 | 3 | `*.backup`, `*.before-*` |
| 根目录临时脚本 | 15+ | `apply-*.js`, `fix-*.js`, `create-*.py` 等 |

### 3. 精简启动脚本

| 操作 | 文件 | 原因 |
|------|------|------|
| 删除 | `start-dev-en.ps1` | 与 `start-dev.ps1` 功能重复 |
| 删除 | `clean-and-start.bat` | 与 `start-dev.bat` 功能重复 |
| 删除 | `stop-dev.bat` | 与 `kill-node.bat` 功能重复 |
| 删除 | `cleanup-backend-new.bat` | 一次性清理脚本，目标目录已不存在 |
| 保留 | `start-dev.ps1` | 完整 PowerShell 启动脚本 |
| 保留 | `start-dev.bat` | 完整 Batch 启动脚本 |
| 保留 | `kill-node.ps1` | 快速停止 Node |
| 保留 | `kill-node.bat` | 快速停止 Node |

### 4. 增强 `.gitignore`

新增 8 类排除规则，防止未来同类文件污染仓库：

- 根目录临时诊断报告 (`/*_REPORT.md`, `/TASK_*_ANALYSIS.md` 等)
- 临时分析脚本 (`code/backend/analyze-*.js`, `diagnose-*.js` 等)
- xlsx 数据文件 (`code/backend/data/csv/*.xlsx`)
- 备份文件 (`*.backup`, `*.before-*`)
- 根目录临时脚本 (`/apply-*.js`, `/fix-*.js` 等)

---

## 效果

| 指标 | 清理前 | 清理后 |

| 启动脚本 | 8 个 | 4 个 |
| 泄露数据文件 | 80+ xlsx | 0 |
