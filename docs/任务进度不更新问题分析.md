**触发场景：**
1. 用户打开任务监控页面 → WebSocket订阅任务
2. 用户切换到其他页面（如文件管理、智能分析）→ TaskMonitor组件销毁
3. `onUnmounted` 钩子触发 → 调用 `unsubscribeTask`
4. WebSocket发送取消订阅消息 → 后端停止推送进度
5. 即使用户切回任务监控页面，也需要重新建立连接

**为什么会多次断开？**
- Vue Router在路由切换时会销毁旧组件
- 如果用户在多个页面间频繁切换，会触发多次订阅/取消订阅
- 日志显示该任务经历了3次不同的Socket连接

### ✅ **已实施的修复：延迟取消订阅**

**修改文件**: [`TaskMonitor.vue`](file://d:\AICODEING\aitraining\code\frontend\src\views\crawler\TaskMonitor.vue#L120-L135)

**修复逻辑：**
```typescript
// ✅ 修复后的代码
let isComponentMounted = true

onUnmounted(() => {
  isComponentMounted = false
  
  // 只有在任务已完成/失败/中止时才取消订阅
  const task = crawlerStore.tasks.find(t => t.id === taskId)
  const shouldUnsubscribe = !task || 
                           ['completed', 'failed', 'cancelled', 'stopped'].includes(task.status)
  
  if (shouldUnsubscribe) {
    console.log('[TaskMonitor] ✅ 任务已结束，取消WebSocket订阅')
    crawlerStore.unsubscribeTask(taskId)
  } else {
    console.log('[TaskMonitor] ⚠️ 任务仍在运行，保持WebSocket连接 (status:', task?.status, ')')
    // 不取消订阅，让Socket继续接收进度更新
  }
})
```

**修复效果：**
- ✅ 用户切换到其他页面时，WebSocket保持连接
- ✅ 后端继续推送进度更新
- ✅ 用户切回任务监控页面时，能看到最新的进度
- ✅ 只有任务真正结束时才断开连接

### 🚀 **可选优化方案**

#### **方案B：实现全局WebSocket管理器**

创建一个全局的WebSocket Store，独立于组件生命周期：

```typescript
// stores/websocket.ts
export const useWebSocketStore = defineStore('websocket', {
  state: () => ({
    socket: null as any,
    subscribedTasks: new Set<string>(),
    reconnectAttempts: 0
  }),
  
  actions: {
    init() {
      if (this.socket) return
      
      this.socket = io(import.meta.env.VITE_API_URL)
      
      // 自动重连机制
      this.socket.on('disconnect', () => {
        console.warn('[WebSocket] 连接断开，尝试重连...')
        this.reconnect()
      })
      
      this.socket.on('connect', () => {
        console.log('[WebSocket] 连接成功')
        this.reconnectAttempts = 0
        
        // 重新订阅所有任务
        this.subscribedTasks.forEach(taskId => {
          this.socket.emit('subscribe', { taskId })
        })
      })
    },
    
    subscribeTask(taskId: string) {
      this.init()
      this.socket.emit('subscribe', { taskId })
      this.subscribedTasks.add(taskId)
    },
    
    unsubscribeTask(taskId: string) {
      this.socket?.emit('unsubscribe', { taskId })
      this.subscribedTasks.delete(taskId)
      
      // 如果没有订阅的任务了，可以断开连接
      if (this.subscribedTasks.size === 0) {
        this.socket?.disconnect()
        this.socket = null
      }
    },
    
    reconnect() {
      if (this.reconnectAttempts >= 5) {
        console.error('[WebSocket] 重连失败，已达最大尝试次数')
        return
      }
      
      this.reconnectAttempts++
      const delay = Math.min(1000 * Math.pow(2, this.reconnectAttempts), 30000)
      
      setTimeout(() => {
        this.socket?.connect()
      }, delay)
    }
  }
})
```

**优势：**
- WebSocket连接与应用生命周期绑定，而非组件生命周期
- 支持自动重连
- 多标签页共享同一个连接
