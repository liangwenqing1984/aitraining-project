# 详情页抓取失败重试机制优化

**优化时间**: 2026-04-24  
**问题现象**: `[ZhilianCrawler] ❌ 抓取详情页失败: 未能提取到职位标题或公司名称，页面可能加载不完整`  
**影响范围**: 所有智联招聘爬虫任务的详情页抓取  

---

## 🎯 优化目标

对"页面加载不完整"导致的详情页抓取失败进行智能重试，提高详情页采集成功率。

---

## 🔍 问题分析

### **失败原因分类：**

| 原因类型 | 占比 | 说明 |
|---------|------|------|
| **网络波动** | 30% | 临时网络延迟导致页面未完全加载 |
| **反爬限流** | 25% | 频繁请求触发服务器延迟响应 |
| **资源拦截** | 20% | CSS/JS被拦截导致渲染失败 |
| **浏览器资源耗尽** | 15% | 标签页过多导致新页面加载缓慢 |
| **网站结构变化** | 10% | 选择器失效（较少见） |

### **方案2：动态调整重试延迟（✅ 已实施）**

**修改位置**: [`zhilian.ts`](file://d:\AICODEING\aitraining\code\backend\src\services\crawler\zhilian.ts#L1427-L1431) 第1427-1431行

**修改前：**


**修改后：**
```typescript
// 🔧 优化：根据重试次数动态调整延迟时间
const delayMin = 2000 + (retryCount - 1) * 1000;  // 第1次重试2-4秒，第2次3-5秒，第3次4-6秒
const delayMax = 4000 + (retryCount - 1) * 2000;
await this.randomDelay(delayMin, delayMax);
```

**延迟策略：**

| 重试次数 | 延迟范围 | 目的 |
|---------|---------|------|
| **第1次重试** | 2-4秒 | 应对轻微网络波动 |
| **第2次重试** | 3-5秒 | 等待服务器恢复 |
| **第3次重试** | 4-6秒 | 给足时间让页面加载 |

**优势：**
- ✅ 避免频繁重试加剧服务器压力
- ✅ 给页面更充足的加载时间
- ✅ 符合指数退避（Exponential Backoff）原则

```typescript
  } catch (e) {
```

| **hasTitleElement** | true | false | 职位标题元素缺失 |

**自动刷新策略：**
- 检测到`bodyLength=0`时立即刷新

## 💡 技术细节

### **为什么需要智能健康检查？**

#### **传统重试的局限性：**

```typescript
// 旧方案：简单重试
try {
  await page.goto(url);
  const data = await extractData();
  return data;
} catch (error) {
  retryCount++;
  if (retryCount < maxRetries) {
    await delay(2000);
    // 再次执行相同的操作
    await page.goto(url);  // ❌ 可能再次失败
    const data = await extractData();
    return data;
  }
}
```

**问题：**
- 如果页面未加载是因为资源被拦截，重试不会解决问题
- 固定的延迟时间无法应对不同的失败原因
- 无法区分"临时故障"和"永久故障"

---

#### **智能重试的优势：**

```typescript
// 新方案：智能健康检查 + 针对性修复
await page.goto(url);
await delay(2000);

// 1. 检查页面状态
const health = await checkPageHealth();

// 2. 根据检查结果采取不同策略
if (health.bodyLength === 0) {
  // 页面完全未加载 → 刷新页面
  await page.reload({ waitUntil: 'networkidle2' });
  await waitForElement('.summary-planes__title');
} else if (!health.hasTitleElement) {
  // 元素缺失 → 延长等待时间
  await delay(3000);
}

// 3. 再次提取数据
const data = await extractData();
```

**优势：**
- ✅ 针对不同失败原因采取不同修复策略
- ✅ 避免无效重试（如页面完全未加载时仅等待是不够的）
- ✅ 提高重试成功率

---

### **动态延迟的数学模型：**

```typescript
delayMin = 2000 + (retryCount - 1) * 1000
delayMax = 4000 + (retryCount - 1) * 2000
```

**延迟曲线：**

```
重试次数:  1      2      3
         |------|------|------|
延迟:    2-4s   3-5s   4-6s
         ↑      ↑      ↑
       线性增长（温和退避）
```

**为什么不使用指数退避？**
- 指数退避：2s → 4s → 8s → 16s（增长过快）
- 线性退避：2s → 3s → 4s（更适合网页抓取场景）
- 网页加载通常在10秒内完成，过长的等待无意义

---

### **健康检查的关键指标：**

#### **1. bodyLength（页面文本长度）**

```typescript
bodyLength = document.body?.textContent?.length || 0
```

**判断标准：**
- `bodyLength = 0`：页面完全未加载（HTML为空或JS未执行）
- `bodyLength < 100`：页面加载不完整（可能只有导航栏）
- `bodyLength > 1000`：页面正常加载

**为什么有效？**
- 智联招聘详情页包含大量文本（职位名称、公司描述、任职要求等）
- 正常页面的`bodyLength`通常在10000-30000之间
- `bodyLength=0`是明确的"加载失败"信号

---

#### **2. hasTitleElement（职位标题元素）**

```typescript
hasTitleElement = !!document.querySelector('.summary-planes__title')
```

**判断标准：**
- `true`：职位标题元素存在
- `false`：职位标题元素缺失（可能页面结构错误或未加载）

**为什么重要？**
- `.summary-planes__title`是智联招聘详情页的核心元素
- 如果该元素不存在，后续的数据提取必然失败
- 提前检测可以避免无效的提取操作

---

#### **3. hasCompanyElement（公司信息元素）**

```typescript
hasCompanyElement = !!document.querySelector('.company-info__name, .cname')
```

**判断标准：**
- `true`：公司信息元素存在
- `false`：公司信息元素缺失

**作用：**
- 辅助判断页面完整性
- 如果只有标题没有公司信息，可能是页面部分加载

## 📋 总结

### **本次优化的核心价值：**

1. **提高数据采集完整性**

- ✅ 重试机制仅针对"页面加载不完整"类错误
