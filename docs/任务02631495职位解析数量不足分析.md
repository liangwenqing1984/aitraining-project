## 🔍 根本原因分析

### **原因1：智联招聘列表页动态加载机制（最可能）**

#### **现象：**
- 智联招聘采用**懒加载（Lazy Loading）**技术
- 初始HTML只包含部分职位（通常15-18个）
- 滚动页面后才加载剩余职位
- 爬虫的等待策略可能不足以触发所有职位加载

#### **证据链：**

```typescript
// zhilian.ts 中的等待策略
await new Promise(resolve => setTimeout(resolve, 2000));  // 仅等待2秒

// 滚动策略
for (let i = 0; i < 5; i++) {
  window.scrollBy(0, window.innerHeight);
  await new Promise(resolve => setTimeout(resolve, 1000));
}
```

**问题：**
- 2秒初始等待可能不够（网络慢时）
- 滚动5次×1秒=5秒，可能仍不足以加载所有职位
- 智联招聘可能有更复杂的加载逻辑（如Intersection Observer）

---

### **原因2：DOM选择器匹配不完整**

#### **当前使用的选择器策略：**

**策略1：`jobinfo` 标签**
```javascript
const jobInfoElements = Array.from(document.querySelectorAll('jobinfo'));
```

**策略2：职位卡片容器**
```javascript
const cardSelectors = [
  '.positionlist__list .joblist-box__item',  // ✅ 优先
  '.job-list-box .job-card-wrapper',
  '.joblist-box__item',

  '[class*="position-item"]',
  '.search-result-list > div',
  'article[class*="job"]',
  'section[class*="job"]'

```

**策略3：职位链接**
```javascript
const jobLinks = Array.from(document.querySelectorAll('a[href*="/jobdetail/"], a[href*="/job/"]'));
```

#### **可能的问题：**

1. **策略1未命中**
   - 如果页面没有`<jobinfo>`自定义标签，策略1返回0个
   - 依赖策略2和策略3补充

2. **策略2的选择器不匹配**
   - 智联招聘可能更新了CSS类名
   - `.positionlist__list .joblist-box__item` 可能不是唯一结构
   - 某些职位可能使用不同的DOM结构（如广告位、推荐位）

3. **策略3的去重逻辑过于严格**
   ```javascript
   if (globalSeenTitles.has(title)) return;  // 标题去重
   if (globalSeenHrefs.has(href)) return;    // 链接去重
   ```
   - 如果两个职位标题相同（罕见但可能），只会保留第一个
   - 如果链接格式不一致（http vs https），可能误判为不同职位

---

### **原因3：职位卡片渲染延迟**

#### **现代前端框架的特性：**

智联招聘使用React/Vue等框架，职位卡片的渲染可能存在：

1. **虚拟滚动（Virtual Scrolling）**
   - 只渲染可视区域内的职位
   - 滚动时才创建新的DOM节点
   - 爬虫的滚动可能不够平滑或速度过快

2. **异步组件加载**
   - 职位卡片可能是异步组件
   - 需要等待组件挂载完成
   - `domcontentloaded`事件触发时，组件可能还未完全渲染

3. **条件渲染**
   - 某些职位可能需要用户交互后才显示
   - 如"查看更多"按钮需要点击

---

### **原因4：反爬机制干扰**

#### **可能的反爬措施：**

1. **请求频率限制**
   - 连续访问多页可能被限流
   - 第2页比第1页少5个职位，可能是限流导致

2. **JavaScript挑战**
   - 某些职位可能需要执行特定JS才能显示
   - Puppeteer可能被检测为非真实浏览器

3. **动态HTML结构**
   - 每次请求的HTML结构可能略有不同
   - 防止基于固定选择器的爬虫
