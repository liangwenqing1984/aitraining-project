## 📊 技术细节

### **为什么需要设置taskId？**

#### **设计模式：依赖注入 vs 全局状态**

**当前实现（混合模式）：**
- [io](file://d:\AICODEING\aitraining\code\backend\src\app.ts#L16-L16): 通过ES模块导入（全局单例）
- [taskId](file://d:\AICODEING\aitraining\code\frontend\src\views\crawler\TaskMonitor.vue#L179-L179): 通过实例属性传递（需要手动设置）

**更好的设计（纯依赖注入）：**

```typescript
// 方案1：构造函数注入
class ZhilianCrawler {
  private taskId: string;
  
  constructor(taskId: string) {
    this.taskId = taskId;
  }
}

// 使用
const crawler = new ZhilianCrawler(taskId);
```

```typescript
// 方案2：方法参数传递
async *crawl(config: TaskConfig, signal: AbortSignal, taskId: string): AsyncGenerator<JobData> {
  // 直接使用taskId参数
}

// 使用
for await (const job of crawler.crawl(config, signal, taskId)) {
  // ...
}
```

**当前选择的权衡：**
- ✅ 优点：最小化代码改动，快速修复
- ⚠️ 缺点：使用`(crawler as any).taskId`绕过类型检查，不够优雅
- 💡 建议：后续重构时使用构造函数注入

### **多组合任务的进度计算策略**

根据《长运行任务进度管理与同步规范》：

| 场景 | 计算公式 | 示例 |
|------|---------|------|
| **组合1/4完成** | `(1 / 4) × 99 = 25%` | 25% |
| **组合2/4完成** | `(2 / 4) × 99 = 50%` | 50% |
| **组合3/4完成** | `(3 / 4) × 99 = 74%` | 74%（四舍五入） |
| **组合4/4完成** | `(4 / 4) × 99 = 99%` | 99% |
| **任务全部完成** | TaskService设置为100% | 100% |

**为什么乘以99而不是100？**
- 预留1%给TaskService在任务完全结束时设置
- 避免爬虫和TaskService同时设置100%造成竞态条件
