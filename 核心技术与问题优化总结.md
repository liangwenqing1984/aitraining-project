### 2.2 设计模式应用

#### （1）策略模式 - 多平台爬虫抽象

**问题背景：**
不同招聘平台（智联、前程无忧）的HTML结构、API接口、反爬策略完全不同，需要统一的调用接口。

**实现方案：**
```typescript

      // 解析逻辑

// 前程无忧策略
class Job51Crawler implements CrawlerStrategy {
  async crawl(keyword: string, city: string, maxPages: number) {
    // 前程无忧特有逻辑
    const url = `https://search.51job.com/list/?keyword=${keyword}&city=${city}`;
    // ...
  }
  
  parsePage(html: string) {
    // 前程无忧特有选择器
```

#### （2）观察者模式 - WebSocket实时推送

**问题背景：**
爬虫任务可能持续数分钟，用户需要实时看到进度和日志，传统HTTP轮询效率低且延迟高。

**实现方案：**
```typescript
// 后端：Socket.IO服务端
import { Server } from 'socket.io';

const io = new Server(httpServer, {
  cors: { origin: '*' }
});

// 任务启动时建立连接
io.on('connection', (socket) => {
  console.log('Client connected:', socket.id);
  
  // 加入任务房间
  socket.on('join-task', (taskId: string) => {
    socket.join(`task:${taskId}`);
  });
  
  // 断开连接
  socket.on('disconnect', () => {
    console.log('Client disconnected:', socket.id);
  });
});

// 爬虫服务中推送进度
class TaskService {
  async startTask(taskId: string) {
    const crawler = CrawlerFactory.create(task.platform);
    
    crawler.on('progress', (current, total, logs) => {
      // 推送进度到前端
      io.to(`task:${taskId}`).emit('task-progress', {
        taskId,
        current,
        total,
        percentage: Math.round((current / total) * 100),
        logs
      });
    });
    
    await crawler.crawl(task.keyword, task.city, task.maxPages);
  }
}

// 前端：Socket.IO客户端
import { io } from 'socket.io-client';

const socket = io('http://localhost:3004');

// 加入任务房间
socket.emit('join-task', taskId.value);

// 监听进度更新
socket.on('task-progress', (data) => {
  progress.value = data.percentage;
  logs.value.push(...data.logs);
  
  // 自动滚动到底部
  nextTick(() => {
    logContainer.value.scrollTop = logContainer.value.scrollHeight;
  });
});
```

**优势：**
- ✅ 延迟<100ms，用户体验极佳
- ✅ 服务器主动推送，减少无效请求
- ✅ 支持多客户端同时监听同一任务

#### （3）工厂模式 - 动态创建爬虫实例

**问题背景：**
根据任务配置中的平台字段，动态创建对应的爬虫实例，避免大量if-else判断。

**实现方案：**
```typescript
class CrawlerFactory {
  private static crawlers = new Map<string, CrawlerStrategy>();
  
  static create(platform: string, config?: CrawlerConfig): CrawlerStrategy {
    if (!this.crawlers.has(platform)) {
      switch (platform) {
        case 'zhilian':
          this.crawlers.set(platform, new ZhilianCrawler(config));
          break;
        case 'job51':
          this.crawlers.set(platform, new Job51Crawler(config));
          break;
        default:
          throw new Error(`Unsupported platform: ${platform}`);
      }
    }
    return this.crawlers.get(platform)!;
  }
  
  // 清除缓存（用于测试或配置变更）
  static clearCache() {
    this.crawlers.clear();
  }
}

// 使用示例
const crawler = CrawlerFactory.create(task.platform, {
  headless: true,
  timeout: 30000
});
```

**优势：**
- ✅ 对象创建与使用分离
- ✅ 支持单例缓存提升性能
- ✅ 易于单元测试（可注入Mock对象）

#### （4）单例模式 - 数据库连接池

**问题背景：**
每次请求都创建新的数据库连接会导致资源浪费和性能下降。

**实现方案：**
```typescript
import { Pool } from 'pg';

class DatabasePool {
  private static instance: Pool;
  
  private constructor() {}
  
  static getInstance(): Pool {
    if (!DatabasePool.instance) {
      DatabasePool.instance = new Pool({
        host: process.env.DB_HOST || 'localhost',
        port: parseInt(process.env.DB_PORT || '5432'),
        database: process.env.DB_NAME || 'training_exercises',
        user: process.env.DB_USER || 'liangwenqing',
        password: process.env.DB_PASSWORD || 'liangwenqing',
        max: 20, // 最大连接数
        idleTimeoutMillis: 30000,
        connectionTimeoutMillis: 2000
      });
      
      // 连接错误处理
      DatabasePool.instance.on('error', (err) => {
        console.error('Unexpected error on idle client', err);
        process.exit(-1);
      });
    }
    return DatabasePool.instance;
  }
  
  // 查询封装
  static async query(text: string, params?: any[]) {
    const pool = this.getInstance();
    const start = Date.now();
    const res = await pool.query(text, params);
    const duration = Date.now() - start;
    console.log('Executed query', { text, duration, rows: res.rowCount });
    return res;
  }
}

// 使用示例
const result = await DatabasePool.query(
  'SELECT * FROM tasks WHERE id = $1',
  [taskId]
);
```

**优势：**
- ✅ 全局唯一连接池，避免资源泄漏
- ✅ 连接复用提升性能
- ✅ 统一管理连接生命周期

## 三、核心技术难点与解决方案

### 3.1 难点1：动态网页爬虫稳定性

#### 问题描述

1. **DOM结构频繁变化**：招聘网站更新前端框架导致CSS选择器失效
2. **JavaScript渲染延迟**：页面加载完成后仍需等待JS执行才能获取完整数据
3. **反爬策略干扰**：IP封禁、验证码、请求频率限制
4. **详情页跳转复杂**：列表页仅显示摘要，需逐个跳转详情页获取完整信息

#### 根本原因分析

- 招聘网站采用React/Vue等现代前端框架，DOM节点动态生成
- 为防止爬虫，网站故意混淆class名称（如`class="job-info"`变为`class="j-i_abc123"`）
- Puppeteer默认超时时间过短，未等待AJAX请求完成

#### 解决方案

**方案1：多重选择器降级策略**

```typescript

  }
}
```

### 3.2 难点2：城市代码精准映射

#### 问题描述

1. **初期误解**：认为同一省份的城市共享区域代码（如黑龙江省所有城市共用622）
2. **实际发现**：每个城市有独立的3位数字代码（哈尔滨622、齐齐哈尔623、大庆627...）
3. **覆盖范围**：全国数百个城市需要逐一验证和映射
4. **动态变化**：网站可能不定期调整城市代码

#### 根本原因分析

- 官方文档未公开城市代码映射规则
- 前端JS文件中代码经过压缩和混淆
- 部分城市存在别名（如"呼和浩特"vs"呼和浩特市"）

#### 解决方案

```typescript
class ZhilianCrawler {
  async getCityCode(cityName: string): Promise<string> {
    // 精确匹配
    if (ZHILIAN_CITY_CODES[cityName]) {
      return ZHILIAN_CITY_CODES[cityName];
    }
    
    // 去除后缀再匹配（如"哈尔滨市" -> "哈尔滨"）
    const simplified = cityName.replace(/[市县区省市自治区]/g, '');
    if (ZHILIAN_CITY_CODES[simplified]) {
      return ZHILIAN_CITY_CODES[simplified];
    }
    
    // 模糊匹配（包含关系）
    for (const [name, code] of Object.entries(ZHILIAN_CITY_CODES)) {
      if (name.includes(simplified) || simplified.includes(name)) {
        console.warn(`⚠️ 城市名称模糊匹配: ${cityName} -> ${name} (${code})`);
        return code;
      }
    }
    
    throw new Error(`未找到城市代码: ${cityName}`);
  }
}
```

### 3.3 难点3：CSV数据一致性保障

#### 问题描述

1. **列错位**：表头定义与数据写入顺序不一致导致数据错乱
2. **特殊字符破坏格式**：职位名称包含逗号（如"Java工程师,高级"）导致CSV解析错误
3. **Excel中文乱码**：缺少UTF-8 BOM头，Excel无法识别编码
4. **引号嵌套问题**：企业名称包含引号（如"阿里巴巴"集团"）

#### 根本原因分析

- 早期实现使用字符串拼接而非专业CSV库
- 未考虑边界情况（空值、特殊字符、换行符）
- 不同操作系统对换行符处理不同（Windows `\r\n` vs Unix `\n`）

#### 解决方案

### 3.4 难点4：OAuth2回调路径冲突

#### 问题描述

1. **路由冲突**：Auth服务回调地址必须是`/api/auth/callback`，但前端也有`/callback`路由
2. **State参数校验失败**：前后端state不一致导致认证失败
3. **Token存储安全**：localStorage易受XSS攻击
4. **登出流程复杂**：需要先获取logoutTicket再跳转认证中心

#### 根本原因分析

- OAuth2规范要求redirect_uri必须精确匹配
- 前端SPA路由与后端API路由在同一域名下容易冲突
- Cookie的httpOnly标志与JavaScript读取需求矛盾

#### 解决方案

**方案1：后端统一处理OAuth2回调**

```typescript
// code/backend/src/controllers/authController.ts
import { Router, Request, Response } from 'express';
import authService from '../services/authService';

const router = Router();

// OAuth2回调接口（Auth服务会POST到此）
router.get('/callback', async (req: Request, res: Response) => {
  const { code, state, error } = req.query;
  
  // 1. 检查错误
  if (error) {
    console.error('OAuth2授权失败:', error);
    return res.redirect(`http://localhost:3000/login?error=${error}`);
  }
  
  // 2. 校验state（防止CSRF攻击）
  const sessionState = req.session.oauthState;
  if (state !== sessionState) {
    console.error('State校验失败');
    return res.redirect('http://localhost:3000/login?error=invalid_state');
  }
  
  try {
    // 3. 用code换取Token
    const tokenResult = await authService.exchangeCodeForToken(code as string);
    
    // 4. 获取用户信息
    const userInfo = await authService.getUserInfo(tokenResult.data.accessToken);
    
    // 5. 将Token保存到Cookie
    authService.setTokenToCookie(res, tokenResult.data);
    
    // 6. 保存用户信息到session
    req.session.userInfo = userInfo;
    
    // 7. 重定向到前端首页
    res.redirect('http://localhost:3000/');
    
  } catch (error) {
    console.error('Token交换失败:', error);
    res.redirect('http://localhost:3000/login?error=token_exchange_failed');
  }
});

export default router;
```

**方案2：Cookie Token存储策略**

```typescript
// code/backend/src/services/authService.ts
import { Response } from 'express';

interface TokenData {
  accessToken: string;
  refreshToken: string;
  tokenType: string;
  expiresIn: number;
}

class AuthService {
  /**
   * 将Token保存到Cookie
   * 
   * 安全策略：
   * - access_token: httpOnly=false（需JS读取添加到请求头）
   * - refresh_token: httpOnly=true（更安全，仅后端可访问）
   */
  setTokenToCookie(res: Response, tokenData: TokenData) {
    const isProduction = process.env.NODE_ENV === 'production';
    
    // Access Token Cookie
    res.cookie('oauth_access_token', tokenData.accessToken, {
      httpOnly: false,  // 允许JS读取
      secure: isProduction,  // 生产环境仅HTTPS
      sameSite: 'lax',
      maxAge: tokenData.expiresIn * 1000,
      path: '/'
    });
    
    // Refresh Token Cookie（更安全）
    res.cookie('oauth_refresh_token', tokenData.refreshToken, {
      httpOnly: true,   // 禁止JS读取
      secure: isProduction,
      sameSite: 'strict',
      maxAge: 30 * 24 * 60 * 60 * 1000,  // 30天
      path: '/api/auth/refresh-token'  // 仅刷新接口可访问
    });
    
    // Token Type
    res.cookie('oauth_token_type', tokenData.tokenType, {
      httpOnly: false,
      secure: isProduction,
      sameSite: 'lax',
      maxAge: tokenData.expiresIn * 1000,
      path: '/'
    });
    
    // Expires In
    res.cookie('oauth_expires_in', tokenData.expiresIn.toString(), {
      httpOnly: false,
      secure: isProduction,
      sameSite: 'lax',
      maxAge: tokenData.expiresIn * 1000,
      path: '/'
    });
    
    console.log('✅ Token已保存到Cookie');
  }
  
  /**
   * 从Cookie清除Token
   */
  clearTokenCookie(res: Response) {
    res.clearCookie('oauth_access_token');
    res.clearCookie('oauth_refresh_token');
    res.clearCookie('oauth_token_type');
    res.clearCookie('oauth_expires_in');
    res.clearCookie('oauth_user_info');
    
    console.log('✅ Token Cookie已清除');
  }
}

export default new AuthService();
```

```typescript
    }
```

```typescript
class AnalysisService {
  /**
   * 流式分析CSV文件（内存占用恒定）
   */
  async analyzeCSVStream(filePath: string) {
    return new Promise((resolve, reject) => {
      const stats = {
        totalRecords: 0,
        salaryData: [],
        cityDistribution: {},
        educationDistribution: {},
        // ... 其他统计字段
      };
      
      fs.createReadStream(filePath)
        .pipe(csvParser())
        .on('data', (row) => {
          // 逐行处理，不累积原始数据

          const salary = this.parseSalary(row['薪资范围']);

          }

          const city = row['工作地点'];
          if (city) {
            stats.cityDistribution[city] = 
              (stats.cityDistribution[city] || 0) + 1;
          }
          
          // 统计学历分布
          const education = row['学历要求'];
          if (education) {
            stats.educationDistribution[education] = 
              (stats.educationDistribution[education] || 0) + 1;
          }
          
          // 每处理1000条输出进度
          if (stats.totalRecords % 1000 === 0) {
            console.log(`已处理${stats.totalRecords}条记录...`);
          }
        })
        .on('end', () => {
          console.log(`✅ 分析完成: ${stats.totalRecords}条记录`);
          
          // 计算聚合指标
          const result = this.calculateAggregates(stats);
          resolve(result);
        })
        .on('error', reject);
    });
  }

   * 计算聚合指标

      ? stats.salaryData.reduce((a, b) => a + b, 0) / stats.salaryData.length

      .sort(([,a]: any, [,b]: any) => b - a)
      .slice(0, 10)
      .map(([city, count]) => ({
        name: city,
        value: count,
        percentage: ((count as number) / stats.totalRecords * 100).toFixed(2)
      }));
    
    return {
      totalRecords: stats.totalRecords,
      avgSalary: Math.round(avgSalary),
      topCities,
      educationDistribution: stats.educationDistribution,
      // ... 其他指标
    };
```

**方案3：优化薪资解析正则表达式**

```typescript
/**
 * 解析薪资格式（优化版）
 * 
 * 支持格式：
 * - "10K-15K"
 * - "10000-15000"
 * - "10-15K"
 * - "1万-1.5万"
 * - "10K以上"
 * - "面议"
 */
function parseSalary(salaryText: string): number | null {
  if (!salaryText || salaryText === '面议') {
    return null;
  }
  
  // 优化后的正则（避免回溯）
  // 使用原子组和非捕获组提升性能
  const patterns = [
    // 匹配 "10K-15K" 或 "10-15K"
    /(\d+(?:\.\d+)?)\s*[Kk]\s*[-~至到]\s*(\d+(?:\.\d+)?)\s*[Kk]?/,
    
    // 匹配 "1万-1.5万"
    /(\d+(?:\.\d+)?)\s*万\s*[-~至到]\s*(\d+(?:\.\d+)?)\s*万/,
    
    // 匹配 "10000-15000"
    /(\d{4,})\s*[-~至到]\s*(\d{4,})/,
    
    // 匹配 "10K以上"
    /(\d+(?:\.\d+)?)\s*[Kk]\s*以上/
  ];
  
  for (const pattern of patterns) {
    const match = salaryText.match(pattern);
    if (match) {
      let min = parseFloat(match[1]);
      let max = parseFloat(match[2] || match[1]);
      
      // 单位转换
      if (salaryText.includes('万')) {
        min *= 10000;
        max *= 10000;
      } else if (salaryText.toLowerCase().includes('k')) {
        min *= 1000;
        max *= 1000;
      }
      
      // 返回平均值
      return Math.round((min + max) / 2);
    }
  }
  
  return null;
}
```

   - 针对性优化而非过早优化

4. **完善的错误处理**
   - 所有异步操作都有try-catch
   - 用户友好的错误提示
   - 详细的日志记录便于排查

---

### 5.2 踩坑记录

1. **不要假设API行为**
   - ❌ 错误：认为智联招聘城市代码是区域共享
   - ✅ 正确：通过实际测试验证每个城市的代码
   
2. **CSV处理必须用专业库**
   - ❌ 错误：字符串拼接导致特殊字符破坏格式
   - ✅ 正确：使用csv-stringify和csv-parser

3. **OAuth2 redirect_uri必须精确匹配**
   - ❌ 错误：前端`/callback`与后端`/api/auth/callback`混用
   - ✅ 正确：统一由后端处理回调

4. **Puppeteer需要充分等待**
   - ❌ 错误：页面加载完立即抓取，遗漏动态内容
   - ✅ 正确：waitForSelector + waitForNetworkIdle

5. **ECharts实例必须手动销毁**
   - ❌ 错误：组件卸载时未dispose导致内存泄漏
   - ✅ 正确：onBeforeUnmount中清理所有图表
